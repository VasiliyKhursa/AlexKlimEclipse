Используем веб-сервисы на примере Twitter

Четвёртый месяц обучения (закрытая зона)

Метки: RESTful, веб-сервисы, Twitter, JSON, AsyncTask, HttpClient, InputStreamReader

Многие популярные сайты имеют собственные веб-сервисы и API для получения нужной информации. Одним из таких примеров может служить Twitter. Мы можем сделать определённый запрос в нужном формате и получить необходимые данные.

В первой части примера мы будем обрабатывать введённый запрос пользователя и пошлём данный запрос на сайт Twitter для поиска, чтобы получить твиты, соответствующие запросу.

Во второй части примера мы получим результат, который будет возвращаться в формате JSON, обработаем его и покажем результат на экране.

Создадим новый проект TwitterService. Сразу, пока не забыли, установим разрешение на использование интернета:


<uses-permission android:name="android.permission.INTERNET"/>

Разработаем дизайн для будущего приложения. Нам понадобится текстовая метка для вывода информации, текстовое поле для ввода запроса, кнопка для отправки запроса и ещё одна текстовая метка для вывода полученного ответа.

Открываем редактор кода для основной активности. Мы можем написать код для щелчка кнопки. Нам нужно получить текст из текстового поля и сохранить его в строковой переменной. Прежде чем посылать запрос на сайт, нужно проверить наличие текста. Если пользователь ничего не вводил в текстовом поле, то не стоит делать лишней работы и гонять данные туда-сюда. Вместо этого мы лучше выведем просьбу ввести текст.

Если пользователь напечатал какой-то текст, то начинаем отправку данных на сервер Твиттера.

Формат Twitter Search API очень прост. Нужно присоединить строку запроса к базовому URL, где query нужно заменить на сам запрос.


http://search.twitter.com/search.json?q=query

Ради интереса, в своём браузере наберите в адресной строке адрес: http://search.twitter.com/search.json?q=pussy, чтобы увидеть, как выглядит ответ сервера. Если вы не робот, то разобраться в данных довольно сложно, их нужно обработать. Если вы пользуетесь Твиттером, то можете в его поисковой строке набрать слово pussy, чтобы увидить результат в более понятном виде.

Не забывайте, что такие символы как пробел и некоторые другие необходимо преобразовать в стандартный html-вид. В Android для этих целей существует метод URLEncoder.encode().


public void onClick(View view) {
	EditText searchTxt = (EditText) findViewById(R.id.search_edit);
	String searchTerm = searchTxt.getText().toString();

	if (searchTerm.length() > 0) {
		try {
			// преобразуем строку в нужный формат
			String encodedSearch = URLEncoder.encode(searchTerm, "UTF-8");
			// формируем строку запроса
			String searchURL = "http://search.twitter.com/search.json?q="
					+ encodedSearch;
			new GetTweetsTask().execute(searchURL);
		} catch (Exception e) {
			tweetDisplay.setText("Whoops - something went wrong!");
			e.printStackTrace();
		}
	} else
		tweetDisplay.setText("Enter a search query!");
}

Теперь разберёмся с получением ответа. Так как операция требует времени на отправку данных и получение ответа, то необходимо реализовать код через AsyncTask.


private class GetTweetsTask extends AsyncTask<String, Void, String> {
}

В нашем случае достаточно двух строковых параметров - первый параметр будет указывать на адрес, по которому нужно обратиться, а третий параметр - на полученный ответ. Второй параметр Void отвечает за изменение прогресса во время выполнения задачи. У нас не используется.

Метод doInBackground()

Начнём с метода doInBackground(). У него будет использоваться параметр строкового типа. Эта строка отвечает за создание адреса Twitter Search API, о котором говорилось выше. Внутри данного метода мы делаем фоновую задачу, которая не должна мешать работе основного потока приложения.

Метод возвращает строковый результат, который соответствует третьему параметру в классе. Он будет использоваться в методе onPostExecute().

Ответ от сервера является строковым значением. Создадим объект StringBuilder. Хотя нам нужен только один адрес, в методе требуется использовать массив строк. Поэтому необходимо использовать цикл, который в реальности будет выполнен один раз. Возможен сценарий, когда действительно нужно использовать массив строк, но нам это сейчас не понадобится.

Для отправки запроса на сервер используется специальный HTTP-клиент. Сам запрос осуществляется при помощи объекта HttpGet (HTTP-команда GET), а ответ возвращается в объекте HttpResponse.

Прежде чем обрабатывать ответ, нужно убедиться, что мы успешно получили ответ. Существует вероятность, что нужный сервер недоступен и тогда ответ будет не совсем таким, который мы ожидаем. При успешном соединение возвращается стандартный код 200. Тогда мы начинаем считывать информацию из потока.

Чтение данных происходит при помощи InputStreamReader и BufferedReader строчка за строчкой и заносится в объект StringBuilder, созданный ранее. Когда чтение данных закончится, то полученная строка из StringBuilder и будет возвращаемым значением для метода.


@Override
protected String doInBackground(String... twitterURL) {
	StringBuilder tweetFeedBuilder = new StringBuilder();
	for (String searchURL : twitterURL) {
		HttpClient tweetClient = new DefaultHttpClient();

		try {
			HttpGet tweetGet = new HttpGet(searchURL);
			HttpResponse tweetResponse = tweetClient.execute(tweetGet);
			StatusLine searchStatus = tweetResponse.getStatusLine();
			if (searchStatus.getStatusCode() == 200) {
				HttpEntity tweetEntity = tweetResponse.getEntity();
				InputStream tweetContent = tweetEntity.getContent();

				InputStreamReader tweetInput = new InputStreamReader(
						tweetContent);
				BufferedReader tweetReader = new BufferedReader(
						tweetInput);
				String lineIn;
				while ((lineIn = tweetReader.readLine()) != null) {
					tweetFeedBuilder.append(lineIn);
				}
			} else
				tweetDisplay.setText("Whoops - something went wrong!");
		} catch (Exception e) {
			tweetDisplay.setText("Whoops - something went wrong!");
			e.printStackTrace();
		}
	}
	return tweetFeedBuilder.toString();
}

Метод onPostExecute()

После того, как мы получили результат от метода doInBackground(), можно обработать его для вывода на экран через метод onPostExecute().

У метода один параметр - полученная строка для дальнейшей обработки. Сама строка содержит текст в JSON-формате. Необходимо распарсить строку для удобного чтения при помощи объекта JSONObject.

JSON-текст состоит из массива строк. Если вернуться к нашему запросу в браузере http://search.twitter.com/search.json?q=pussy и посмотреть на результат, то можно увидеть секцию, дающую начало массиву:


"results":[

Внутри массива содержатся твиты плюс дополнительные данные.

Наша задача - вычленить данный массив по имени result через JSONArray.

Получив нужный массив, нужно через цикл получить отдельные элементы массива. Каждый элемент массива содержит множество информации - дату создания, автора твита и т.п. Мы ограничимся только частью информации для вывода на экран.

К примеру, имя автора твита содержится в ключе from_user, а сам твит в ключе text.

Окончательный результат выводится в текстовой метке.


protected void onPostExecute(String result) {
	StringBuilder tweetResultBuilder = new StringBuilder();

	try {
		JSONObject resultObject = new JSONObject(result);
		JSONArray tweetArray = resultObject.getJSONArray("results");

		for (int t = 0; t < tweetArray.length(); t++) {
			JSONObject tweetObject = tweetArray.getJSONObject(t);
			tweetResultBuilder.append(tweetObject
					.getString("from_user") + ": ");
			tweetResultBuilder.append(tweetObject.get("text") + "\n\n");
		}
	} catch (Exception e) {
		tweetDisplay.setText("Whoops - something went wrong!");
		e.printStackTrace();
	}
	if (tweetResultBuilder.length() > 0)
		tweetDisplay.setText(tweetResultBuilder.toString());
	else
		tweetDisplay
				.setText("Sorry - no tweets found for your search!");
}

Запускаем задачу

В обработчике щелчка кнопки мы уже прописали код для запуска задачи:


new GetTweetsTask().execute(searchURL);

В параметре метода execute() передаётся строка, которую мы использовали в объявлении класса AsyncTask и метода doInBackground.

Наше приложение готово. Запускайте его на эмуляторе или устройстве и тестируйте.

Исходный код:


package ru.alexanderklimov.twitterservice;

import java.io.BufferedReader;
import java.io.InputStream;
import java.io.InputStreamReader;
import java.net.URLEncoder;

import org.apache.http.HttpEntity;
import org.apache.http.HttpResponse;
import org.apache.http.StatusLine;
import org.apache.http.client.HttpClient;
import org.apache.http.client.methods.HttpGet;
import org.apache.http.impl.client.DefaultHttpClient;
import org.json.JSONArray;
import org.json.JSONObject;

import android.os.AsyncTask;
import android.os.Bundle;
import android.app.Activity;
import android.view.Menu;
import android.view.View;
import android.widget.EditText;
import android.widget.TextView;

public class TwitterActivity extends Activity {

	private TextView tweetDisplay;

	@Override
	protected void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);
		setContentView(R.layout.activity_twitter);

		tweetDisplay = (TextView) findViewById(R.id.tweet_txt);

	}

	@Override
	public boolean onCreateOptionsMenu(Menu menu) {
		// Inflate the menu; this adds items to the action bar if it is present.
		getMenuInflater().inflate(R.menu.activity_twitter, menu);
		return true;
	}

	public void onClick(View view) {
		EditText searchTxt = (EditText) findViewById(R.id.search_edit);
		String searchTerm = searchTxt.getText().toString();

		if (searchTerm.length() > 0) {
			try {
				// преобразуем строку в нужный формат
				String encodedSearch = URLEncoder.encode(searchTerm, "UTF-8");
				// формируем строку запроса
				String searchURL = "http://search.twitter.com/search.json?q="
						+ encodedSearch;
				new GetTweetsTask().execute(searchURL);
			} catch (Exception e) {
				tweetDisplay.setText("Whoops - something went wrong!");
				e.printStackTrace();
			}
		} else
			tweetDisplay.setText("Enter a search query!");
	}

	private class GetTweetsTask extends AsyncTask<String, Void, String> {

		@Override
		protected String doInBackground(String... twitterURL) {
			StringBuilder tweetFeedBuilder = new StringBuilder();
			for (String searchURL : twitterURL) {
				HttpClient tweetClient = new DefaultHttpClient();

				try {
					HttpGet tweetGet = new HttpGet(searchURL);
					HttpResponse tweetResponse = tweetClient.execute(tweetGet);
					StatusLine searchStatus = tweetResponse.getStatusLine();
					if (searchStatus.getStatusCode() == 200) {
						HttpEntity tweetEntity = tweetResponse.getEntity();
						InputStream tweetContent = tweetEntity.getContent();

						InputStreamReader tweetInput = new InputStreamReader(
								tweetContent);
						BufferedReader tweetReader = new BufferedReader(
								tweetInput);
						String lineIn;
						while ((lineIn = tweetReader.readLine()) != null) {
							tweetFeedBuilder.append(lineIn);
						}
					} else
						tweetDisplay.setText("Whoops - something went wrong!");
				} catch (Exception e) {
					tweetDisplay.setText("Whoops - something went wrong!");
					e.printStackTrace();
				}
			}
			return tweetFeedBuilder.toString();
		}

		protected void onPostExecute(String result) {
			StringBuilder tweetResultBuilder = new StringBuilder();

			try {
				JSONObject resultObject = new JSONObject(result);
				JSONArray tweetArray = resultObject.getJSONArray("results");

				for (int t = 0; t < tweetArray.length(); t++) {
					JSONObject tweetObject = tweetArray.getJSONObject(t);
					tweetResultBuilder.append(tweetObject
							.getString("from_user") + ": ");
					tweetResultBuilder.append(tweetObject.get("text") + "\n\n");
				}
			} catch (Exception e) {
				tweetDisplay.setText("Whoops - something went wrong!");
				e.printStackTrace();
			}
			if (tweetResultBuilder.length() > 0)
				tweetDisplay.setText(tweetResultBuilder.toString());
			else
				tweetDisplay
						.setText("Sorry - no tweets found for your search!");
		}
	}

}
