Google выпустил обновление к своим картам. Новое API содержит векторные карты, которые быстрее загружаются, а также позволяют пользователям легко переключаться между 2D и 3D режимами, в которых карты можно вращать при помощи жестов.

С новой версией Google Maps Android API v2, разработчики смогут использовать новые возможности:

  • поддержка фрагметов
  • больше слоёв Google Maps, включая спутниковый, гибридный, пробки, а также новый слой — внутренние карты для основных аэропортов и торговых центров
  • уменьшение строк кода, чтобы создавать маркеры и информационные окна (infowindow)

Минимальная требуемая версия — Android 2.2 с OpenGL ES версии 2.

Стоит отметить, что первая версия карт теперь считается устаревшей, и с декабря до марта 2013 у разработчиков есть время на переход на новую версию. Потом вы не сможете получить ключ к карте (старые ключи будут работать).

Теперь можно фрагменты вместо MapView. Для этого достаточно вставить следующий код в разметку:


<fragment
  android:layout_width="match_parent"
  android:layout_height="match_parent"
  class="com.google.android.gms.maps.SupportMapFragment"/>

Во-вторых стало гораздо удобней работать с маркерами. Теперь не нужно городить ItemizedOverlay и тому подобное. Теперь достаточно просто выполнить метод addMarker(MarkerOptions options) с нужными параметрами, и маркер будет красоваться на указанной позиции.

Стало совсем удобно рисовать на карте объекты, которые без проблем будут зумиться и скролиться вместе с картой, без единой заморочки! Достаточно просто добавить фигуру(метод addPolygon(PolygonOptions options)) или линию(метод addPolyline(PolylineOptions options)) на карту, и всё. Это значит, что теперь без проблем можно закрасить страну, город, район. Да всё, что угодно!

У пользователя обязательно должно быть установлено приложение «Карты» от самого Google

Также нужно приложения Сервисы Google Play. Если приложения нет, то пользователю будет сообщено через особое окно.

Для начала нам необходимо установить Google Play services SDK. Запускаем Android SDK Manager и выбираем Extras | Google Play services для установки нужного модуля.

После окончания установки у вас появятся папки android-sdk/extras/google/google_play_services с подпапками docs, libproject и samples. Из содержимого второй папки нужно сделать модуль, который затем нужно подключать к проекту. В третьей папке находятся примеры приложения по работе с картами. Сделайте бекап папок, чтобы всегда иметь чистую версию библиотеки, если будете активно пользоваться примерами.

Получить ключ Maps API v2

Google Maps Android API v2 использует новую систему управления ключами. Старые ключи от Google Maps Android v1 (MapView) не подойдут.

Ключ бесплатен и не имеет ограничений по количеству пользователей вашего приложеиня. Вы можете получить ключ из Google APIs Console.

Ключ Maps API создаётся на основе цифрового сертификата вашего приложения при помощи отпечатка SHA-1, который представляет собой уникальный текст на основе алгоритма SHA-1.

У вас может быть два сертификата: отладочный и релизный.

Отладочный сертификат

Рассмотрим получение отладочного сертификата.

Найдите у себя файл debug.keystore. По умолчанию, он хранится в папке вашего виртуального устройства:

OS X или Linux: ~/.android/

Windows 7: C:\Users\your_user_name\.android\

Вы можете посмотреть расположение вашего файла в Eclipse через меню Windows | Prefs | Android | Build

В Windows 7 запустите команду в командной строке:

keytool -list -v -keystore "C:\Users\your_user_name\.android\debug.keystore" -alias androiddebugkey -storepass android -keypass android

На экране увидите приблизительно такую запись:


Alias name: androiddebugkey
Creation date: Jan 01, 2013
Entry type: PrivateKeyEntry
Certificate chain length: 1
Certificate[1]:
Owner: CN=Android Debug, O=Android, C=US
Issuer: CN=Android Debug, O=Android, C=US
Serial number: 4aa9b300
Valid from: Mon Jan 01 08:04:04 UTC 2013 until: Mon Jan 01 18:04:04 PST 2033
Certificate fingerprints:
     MD5:  AE:9F:95:D0:A6:86:89:BC:A8:70:BA:34:FF:6A:AC:F9
     SHA1: BB:0D:AC:74:D3:21:E1:43:07:71:9B:62:90:AF:A1:66:6E:44:5D:75
     Signature algorithm name: SHA1withRSA
     Version: 3

Найдите строчку, начинающуюся со слов SHA1. Она содержит нужный нам отпечаток SHA-1: двадцать двузначных чисел в шестадцатиричной записи, разделённых двоеточиями.

Релизный ключ

Для получения релизного ключа надо сначала создать подписанное приложение, как вы это делаете для размещеия в Google Play.

Далее в командной строке вводите:


keytool -list -v -keystore <your_keystore_name> -alias <your_alias_name>

<your_keystore_name> - путь к вашем хранилищу (keystore), включая расширение .keystore

<your_alias_name> - псеводоним созданного сертификата.

Идём на страницу Google APIs Console.

Если вы не пользовались консолью Google APIs раньше, то вам будет предложено создать новый проект. Нажимаем на кнопку Create Project. Консоль создаст новый проект API Project. На следующей странице это имя появится в верхней левой части страницы. Нажав на имя проекта, вы можете его переименоваь или сделать другие действия (например, создать новый проект).

Справа вы увидите список доступных сервисов Google. Давайте создадим новый проект для Google Maps Android API, выбираем имя проекта в левой части и щелкаем на кнопке Create....

Выберите новое имя, например, Google Maps Android и нажмите кнопку Create Project. Если список сервисов не отображается, то выберите пункт Services из левой навигационной панели.

Найдите в списке сервисов Google Maps Android API v2 и переключитесь в состояние On. Вам будет показано лицензионное соглашение. Если согласны с ним, то отметьте флажком I agree to these terms и нажимайте на кнопку Accept. Вы вернётесь на предыдущую страницу со списком сервисов. Теперь вы можете получить ключ Maps API.

Если ваше приложение зарегистрировано для использования Google Maps Android API v2 service, вы можете запросить ключ. Можно получать отдельные ключи для каждого проекта.

В левой навигационной панели выбираем API Access. Далее нажимаем Create New Android Key.... Появится диалоговое окно с текстовым полем, в которое нужно вставить отпечаток SHA-1, о котором говорилось выше, затем ввести точку с запятой и имя пакета вашего приложения:


BB:0D:AC:74:D3:21:E1:43:67:71:9B:62:91:AF:A1:66:6E:44:5D:75;ru.alexanderklimov.mapexample

Нажимаем кнопку Create.

В успешном случае вы увидите сам ключ, вставленную строку, дату активации, автора активации.

Полученный ключ вставляем в AndroidManifest.xml внутри секции <application>


<meta-data
    android:name="com.google.android.maps.v2.API_KEY"
    android:value="your_api_key"/>

Далее добавьте необходимое разрешение (замените ru.alexanderklimov.mapexample на свой):


<permission
        android:name="ru.alexanderklimov.mapexample.permission.MAPS_RECEIVE"
        android:protectionLevel="signature"/>
<uses-permission android:name="ru.alexanderklimov.mapexample.permission.MAPS_RECEIVE"/>

Также необходимо добавить следующие разрешения:

  • android.permission.INTERNET
  • com.google.android.providers.gsf.permission.READ_GSERVICES
  • android.permission.WRITE_EXTERNAL_STORAGE

Рекомендуется также добавить два дополнительных разрешения (не обязательно):

  • android.permission.ACCESS_COARSE_LOCATION - позволяет использовать данные от WiFi и сотовых вышек для определения местоположения
  • android.permission.ACCESS_FINE_LOCATION - позволяет использовать данные от GPS для более точного позиционирования

Получаем следующий список:


<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>
<uses-permission android:name="com.google.android.providers.gsf.permission.READ_GSERVICES"/>
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>

Так как Google Maps Android API версии 2 требует OpenGL ES 2, нужно добавить ещё один элемент в манифест, которые предотвратит появление вашего приложения в магазине приложений для старых телефонов:


<uses-feature
  android:glEsVersion="0x00020000"
  android:required="true"/>

Подключаем карты

Когда мы вставили фрагмент с классом com.google.android.gms.maps.MapFragment, то карты уже выводятся на экран. Но если мы хотим управлять картой, добавив некоторую функциональность, то необходимо написать дополнительный код. На данный момент существует две ветки развития фрагментов - встроенная поддержка в новых версиях и поддержка фрагментов в старых версиях через дополнительную библиотеку. В дальнейшем все примеры будут показаны для новых версий (рано или поздно старые версии уйдут, зачем на сайте держать мусор). Но один раз покажу кусочек кода и для старых систем.


private GoogleMap myMap;

@Override
protected void onCreate(Bundle savedInstanceState) {
	super.onCreate(savedInstanceState);
	setContentView(R.layout.activity_main);

	FragmentManager myFragmentManager = getFragmentManager();
	MapFragment myMapFragment = (MapFragment) myFragmentManager
			.findFragmentById(R.id.map);
	myMap = myMapFragment.getMap();
}

Для старых версий код будет отличаться незначительно:


// Класс фрагмента в разметке будет com.google.android.gms.maps.SupportMapFragment
FragmentManager myFragmentManager = getSupportFragmentManager();
SupportMapFragment mySupportMapFragment 
        = (SupportMapFragment)myFragmentManager.findFragmentById(R.id.map);
myMap = mySupportMapFragment.getMap();

Определяем касание на карте

Чтобы определять касание пользователем карты на экране, нужно реализовать интерфейс OnMapClickListener, добавить метод onMapClick() и добавить слушатель setOnMapClickListener().


public class MainActivity extends Activity implements OnMapClickListener{

	@Override
	protected void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);
		setContentView(R.layout.activity_main);
		
		...
		
		myMap.setOnMapClickListener(this);
	}

	@Override
	public void onMapClick(LatLng point) {
		// TODO Auto-generated method stub
		tvInfo.setText(point.toString());
		myMap.animateCamera(CameraUpdateFactory.newLatLng(point));
	}
}

В методе onMapClick() определяются координаты карты в месте касания карты, которые выводятся в текстовой метке. Одновременно точка касания перемещается в центр карты с плавной анимацией при помощи метода animateCamera().

В нашем примере в центре карты оказался финский город Котка - столица финских котов (а может и всей Скандинавии?).

Долгое касание экрана и установка метки

Аналогично обычному касанию экрана, можно обрабатывать долгое касание. Добавляем реализацию интерфейса OnMapLongClickListener, слушатель долгого касания setOnMapLongClickListener() и метод onMapLongClick():


@Override
public void onMapLongClick(LatLng point) {
	// TODO Auto-generated method stub
	tvInfo.setText("Новая метка с координатами " + point.toString());
	myMap.addMarker(new MarkerOptions().position(point).title(
			point.toString()));
}

При долгом касании в указанном месте добавляем новую метку при помощи метода addMarker(). В моём примере добавлены две новые метки рядом с лучшим городом мира Коткой.

Меняем цвет маркеров

Вы можете изменить цвет маркера по умолчанию на другой, добавив .icon(BitmapDescriptor) при создании MarkerOptions.


BitmapDescriptor bitmapDescriptor 
        = BitmapDescriptorFactory.defaultMarker(
            BitmapDescriptorFactory.HUE_AZURE);
  
myMap.addMarker(new MarkerOptions()
        .position(point)
        .icon(bitmapDescriptor)
        .title(point.toString()));

Собственные маркеры

Вы можете не только окрашивать маркер в другой цвет, но и назначить собственный значок для маркера. Добавьте значок в папку drawable и код:


BitmapDescriptor bitmapDescriptor 
        = BitmapDescriptorFactory.fromResource(R.drawable.ic_launcher);
  
myMap.addMarker(new MarkerOptions()
        .position(point)
        .icon(bitmapDescriptor)
        .title(point.toString()));

Определяем касание на маркере и рисуем линии

Раз уж речь идёт о касаниях, то рассмотрим ещё один пример, связанный с определением касания маркера на карте. Для этого нам нужно реализовать интферфейс OnMarkerClickListener, добавить слушатель setOnMarkerClickListener() и написать код для метода onMarkerClick():


boolean markerClicked;
PolylineOptions rectOptions;
Polyline polyline;

@Override
public boolean onMarkerClick(Marker marker) {
	// TODO Auto-generated method stub
	if (markerClicked) {

		if (polyline != null) {
			polyline.remove();
			polyline = null;
		}

		rectOptions.add(marker.getPosition());
		rectOptions.color(Color.RED);
		polyline = myMap.addPolyline(rectOptions);
	} else {
		if (polyline != null) {
			polyline.remove();
			polyline = null;
		}

		rectOptions = new PolylineOptions().add(marker.getPosition());
		markerClicked = true;
	}
	return false;
}

Если приложение распознает касание на маркерах, то между ними будет нарисована красная тонкая линия. На рисунке представлен мистический Коткинский треугольник. Говорят, собаки, которые забредают в этот район, бесследно пропадают. А нечего лезть на помеченную территорию.

Рисуем фигуры

Если простые линии вам не интересны, то используйте Polygon вместо Polyline. В этом случае вы можете получить не просто треугольник из трёх линий, а закрашенный треугольник или другую фигуру. Можно задать отдельно цвет обводки и цвет заливки самой фигуры. Изменения в коде будут незначительными:


PolygonOptions polygonOptions;
Polygon polygon;

@Override
public boolean onMarkerClick(Marker marker) {
	if (markerClicked) {

		if (polygon != null) {
			polygon.remove();
			polygon = null;
		}

		polygonOptions.add(marker.getPosition());
		polygonOptions.strokeColor(Color.RED);
		polygonOptions.fillColor(Color.BLUE);
		polygon = myMap.addPolygon(polygonOptions);
	} else {
		if (polygon != null) {
			polygon.remove();
			polygon = null;
		}

		polygonOptions = new PolygonOptions().add(marker.getPosition());
		markerClicked = true;
	}

	return true;
}

Выводим лицензионное соглашение

При использовании карт Google в своём приложении, вы обязаны включить также показ лицензионного соглашения, например, через пункт меню.

В новых проектах по умолчанию уже используется один пункт меню Settings. Добавим ещё один пункт:


<item
    android:id="@+id/menu_legalnotices"
    android:orderInCategory="100"
    android:showAsAction="never"
    android:title="@string/menu_legalnotices"/>

Добавим строковый ресурс для пункта меню:


<string name="menu_legalnotices">Legal Notices</string>

Вывод соглашения делаем через диалоговое окно:


@Override
public boolean onOptionsItemSelected(MenuItem item) {
	switch (item.getItemId()) {
	case R.id.menu_legalnotices:
		String LicenseInfo = GooglePlayServicesUtil
				.getOpenSourceSoftwareLicenseInfo(getApplicationContext());
		AlertDialog.Builder LicenseDialog = new AlertDialog.Builder(
				MainActivity.this);
		LicenseDialog.setTitle("Legal Notices");
		LicenseDialog.setMessage(LicenseInfo);
		LicenseDialog.show();
		return true;
	}
	return super.onOptionsItemSelected(item);
}

Проверка доступности сервисов Google на устройстве

Рекомендуется проверить, а доступны ли на устройстве сервисы Google, необходимые для работы с картами. Для этого существует метод isGooglePlayServicesAvailable(). Вызовем его в методе активности onResume():


final int RQS_GooglePlayServices = 1;

@Override
protected void onResume() {
	// TODO Auto-generated method stub
	super.onResume();

	int resultCode = GooglePlayServicesUtil
			.isGooglePlayServicesAvailable(getApplicationContext());

	if (resultCode == ConnectionResult.SUCCESS) {
		Toast.makeText(getApplicationContext(),
				"isGooglePlayServicesAvailable SUCCESS", Toast.LENGTH_LONG)
				.show();
	} else {
		GooglePlayServicesUtil.getErrorDialog(resultCode, this,
				RQS_GooglePlayServices);
	}
}

Отключаем ненужные элементы на карте

По умолчанию на карте используются элементы собственного расположения (в верхнем правом углу), компас (в верхнем левом углу при вращении карты) и зум (в нижнем правом углу). Вы можете отключить по отдельности каждый из этих элементов.


myMap.getUiSettings().setZoomControlsEnabled(false);
myMap.getUiSettings().setCompassEnabled(false);
myMap.getUiSettings().setMyLocationButtonEnabled(false);

Жесты

Можно отключить/включить определённые жесты для карты:


myMap.getUiSettings().setRotateGesturesEnabled(true);
myMap.getUiSettings().setScrollGesturesEnabled(true);
myMap.getUiSettings().setTiltGesturesEnabled(true);
myMap.getUiSettings().setZoomGesturesEnabled(true);
// или все сразу
myMap.getUiSettings().setAllGesturesEnabled(true);

Включаем траспортный слой

Достаточно одной таблэтки строчки кода:


myMap.setTrafficEnabled(true);

Настройка карты через XML

Кроме программной настройки карты, можно воспользоваться XML-атрибутами.

Изменим файл разметки, добавив в <fragment> класс "com.google.android.gms.maps.MapFragment" и включив xmlns:map="http://schemas.android.com/apk/res-auto".


<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical"
    tools:context=".MainActivity" >

    <fragment
        xmlns:map="http://schemas.android.com/apk/res-auto"
        android:id="@+id/map"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        class="com.google.android.gms.maps.MapFragment"
        map:cameraBearing="45"
        map:cameraTargetLat="51.507222"
        map:cameraTargetLng="-0.1275"
        map:cameraTilt="30"
        map:cameraZoom="12"
        map:mapType="satellite"
        map:uiCompass="true"
        map:uiRotateGestures="true"
        map:uiScrollGestures="true"
        map:uiTiltGestures="true"
        map:uiZoomControls="true"
        map:uiZoomGestures="true"/>

</LinearLayout>

Теперь, подключаем разметку привычным способом через setContentView() и видим карту с заданными настройками.
