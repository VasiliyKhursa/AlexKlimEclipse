Облака стали ближе

Четвёртый месяц обучения (закрытая зона)

Всё большую популярность набирают облачные вычисления, облачный хостинг и прочие облачные технологии.

Сейчас появилось много компаний, которые предлагают решения, использующие облачные технологии.

Рассмотрим использование облака в Android-приложениях на примере parse.com. Для начала вам следует зарегистрироваться на сайте. На сайте есть обширная документация и примеры (в основном, для iOS). Документация достаточно проста, расскажу вкратце о базовых возможностях.

После регистрации у вас будет своя панель управления, где вы можете просматривать информацию о своих приложениях, а также отсылать push-уведомления.

Установка библиотеки

О том, как устанавливать библиотеку, хорошо показано в видеоматериале. Обратите внимание, что пример связан с котами (наши люди!).

На этой странице вы можете скачать SDK и посмотреть кусок кода для запуска библиотеки.

App

В облаке Parse вы создаёте отдельный App для каждого вашего Android-приложения. Каждый App имеет собственный идентификатор (id) и ключ (client key), которые вы должные использовать в вашем приложении.

Всё, что вам нужно для начала работы, задать ваш идентификатор программы и клиентский ключ в методе onCreate():


public void onCreate() { 
    Parse.initialize(this, "HoEWEtFkj07YwKBjLZfwpOlucLPnH7cHz1DcoD2h", "qYdlWy5fDZu7GuMN7oPhUvDQF82KC5bYuLzwCtcI"); 
}

Объекты ParseObject

Вся работа с данными, которые хранятся в облаке, происходит вокруг объекта ParseObject. Каждый ParseObject содержит пару "ключ-значение" в JSON-формате.

Предположим, вы хотите сохранять лучшие результаты игры. Объект ParseObject может содержать что-то типа:

score: 1337, playerName: "Sean Plott", cheatMode: false

Ключи должны состоять из обычных символов и цифр, а значения могут быть строками, числами булевыми значениями, массивами или объектами - всё, что понимает JSON.

Каждый ParseObject имеет имя класса, чтобы не путать данные между собой. Например, лучшие результаты игры можно сохранить под именем GameScore.

Сохранение данных

Итак, мы определились с именем и данными для нашей игры. Чтобы сохранить данные в облаке, используем следующий код:


ParseObject gameScore = new ParseObject("GameScore");
gameScore.put("score", 1337);
gameScore.put("playerName", "Sean Plott");
gameScore.put("cheatMode", false);
gameScore.saveInBackground();

После выполнения данного кода, данные будут сохранены в облаке. Вы можете убедиться в этом сами, если в панели управления выберите кнопку Data Browser и найдёте свои сохранённые данные на Parse.

Обратите внимание, что класс GameScore будет автоматически создан в облаке при первом упоминании. Сам метод saveInBackground() использует свой поток, не связанный с UI-потоком, что очень важно при работе с данными по сети.

Также вы можете увидеть три дополнительных поля, которые не создавали в коде:

  • objectId - уникальный идентификатор для каждого сохранённого объекта
  • createdAt - время создания объекта
  • updatedAt - время модификации объекта

Можете не обращать на них внимания при работе в своём приложении.

Получение данных

Мы научились сохранять данные в облаке. Теперь рассмотрим обратную задачу - получение данных.

Зная objectId, вы можете получить объект ParseObject через ParseQuery:


ParseQuery query = new ParseQuery("GameScore");
query.getInBackground("xWMyZ4YEGZ", new GetCallback() {
    public void done(ParseObject object, ParseException e) {
        if (e == null) {
        // object will be your game score
    } else {
        // something went wrong
    }
    }
});

Чтобы получить значения из ParseObject, воспользуемся методом getX() для каждого типа данных:


int score = gameScore.getInt("score");
String playerName = gameScore.getString("playerName");
boolean cheatMode = gameScore.getBoolean("cheatMode");

Если вы не знаете тип получаемых данных, то можете использовать общий метод get(key), но тогда вам придётся самому разбираться с типом и приводить к нужному типу.

Недавно обнаружил новый метод fetchInBackground(), который видимо делает ту же работу.

Также есть три специальных значения, которые можно получить при помощи особых методов:


String objectId = gameScore.getObjectId();
Date updatedAt = gameScore.getUpdatedAt();
Date createdAt = gameScore.getCreatedAt();

Вы уже узнали эти служебные значения, о которых упоминалось выше.

Если вы хотите обновить объект, который уже имеется в облаке, то используйте метод refreshInBackground():


myObject.refreshInBackground(new RefreshCallback() {
  public void done(ParseObject object, ParseException e) {
    if (e == null) {
      // Success!
    } else {
      // Failure!
    }
  }
});

Отложенное сохранение

Рассмотренные выше методы сохраняют данные сразу. В случае, если у пользователя возникают проблемы с интернетом, можно использовать метод saveEventually(), который сохранит данные на устройстве и при первой же возможности передаст данные в облако. Если ваше приложение будет закрыто до окончания передачи данных в облако, то при следующем запуске отправка данных будет продолжена.


ParseObject gameScore = new ParseObject("GameScore");
gameScore.put("score", 1337);
gameScore.put("playerName", "Sean Plott");
gameScore.put("cheatMode", false);
gameScore.saveEventually();

Обновление объекта

Обновить данные в объекте очень просто. Просто установите новые значения и вызовите один из нужных методов, например:


// Create the object.
final ParseObject gameScore = new ParseObject("GameScore");
gameScore.put("score", 1337);
gameScore.put("playerName", "Sean Plott");
gameScore.put("cheatMode", false);
gameScore.put("skills", Arrays.asList("pwnage", "flying"));
gameScore.saveInBackground(new SaveCallback() {
  public void done(ParseException e) {
    // Now let's update it with some new data. In this case, only cheatMode and score
    // will get sent to the Parse Cloud. playerName hasn't changed.
    gameScore.put("score", 1338);
    gameScore.put("cheatMode", true);
    gameScore.saveInBackground();
  }
});

Счётчики

В нашем примере есть поле score (Счёт), которые необходимо постоянно обновлять при установлении нового счёта игроком. Мы можем обновлять эти данные, как было показано в примере выше, но это не самый удобный способ. Существуют два метода для увеличения и уменьшения счётчика на единицу следующим образом:


gameScore.increment("score");
gameScore.saveInBackground();

Также можно увеличить на заданную величину через increment(key, amount).

Удаление данных


parseObject.delete();
parseObject.deleteInBackground(); // в отдельном потоке

Работаем с файлами

Доступна работа с файлами, например, с картинками. Размер файла не должен превышать 10 Мб.


// сохраняем изображение
Drawable drawable = ...;
Bitmap bitmap = (Bitmap)((BitmapDrawable) drawable).getBitmap();
ByteArrayOutputStream stream = new ByteArrayOutputStream();
bitmap.compress(Bitmap.CompressFormat.JPEG, 100, stream);
byte[] data = stream.toByteArray();                
ParseFile imageFile = new ParseFile("cat.jpg", data);
imageFile.saveInBackground();

Push-уведомления

Мне лично больше всего привлекло внимание возможность создания push-уведомлений, которыми я активно пользуюсь в программе Справочник про котов, который можно скачать в Google Play.

В самом приложении после инициализации Parse-библиотеки вы должные добавить строчку кода типа:


PushService.subscribe(this, "News", MainActivity.class);

Когда пользователь первый раз запустит ваше приложение, то оно соединится с серверами Parse и зарегистрируется там (при наличии интернета). В моём примере я создал отдельный канал News.

Создать push-уведомление очень просто. Заходите в свою панель управления и выбираете свою программу. На странице программы нажимаете кнопку Push Notifications. У вас откроется специальная страница просмотра отправленных сообщений. По цифрам видно, как растёт число пользователей, которые установили программу и вышли в интернет. Теперь они все могут получать мои сообщения.

Чтобы отправить новое сообщение, нажмите кнопку + Send a push. Откроется новое окно отправки сообщения. Здесь вы можете сформировать своё сообщение, и выбрать режим отправки. В моём случае я выбрал немедленную отправку (Send: Now) и время жизни сообщения (один день).

Пользователи, которые включат интернет после этого срока, моё сообщение уже не увидят. Это полезно, когда вы будет делать новостные рассылки. Нет смысла читать новости недельной давности, особенно если вы будет их выпускать каждый день. Иначе, будет похоже на спам.

После того, как вы определились с настройками, то нажимаете кнопку Send Notification. Обратите внимание, что само приложение не обязательно должно быть запущено на устройстве пользователя, главное - наличие интернета.

Кстати, уведомления приходят не только на реальные устройства, но и на эмулятор. Так что можете тестировать без проблем.

В месяц у вас есть лимит на миллион push-уведомлений. Думаю, вам хватит. За превышение придётся платить.
