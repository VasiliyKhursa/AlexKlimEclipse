Внедряем Google Analytics

Четвёртый месяц обучения (закрытая зона)

Статья основана на версии Google Analytics SDK for Android v2 (Beta). В финальной версии возможны изменения. Раньше использовалась версия 1.0.

Google Analytics - специальный сервис от Google, позволяющий отслеживать действия пользователей. Обычно данный сервис используют веб-мастера на своих сайтах. Но также существует возможность внедрить Google Analytics в приложение при помощи специального SDK.

Основные возможности

  • Позволяет узнать число активных пользователей, использующих ваше приложение
  • В какой точке мира используется приложение
  • Как используются различные возможности в вашем приложении
  • Отслеживание внутренних покупок и транзакций
  • Число и тип краха приложения из-за ошибок
  • Другие полезные параметры

Google Analytics SDK for Android включает в себя классы и методы для слежения за активностю пользователей и создание на основе полученной информации полезной статистики. Статистика не выводится в режиме реального времени, она обрабатывается и выдаётся с суточной задержкой.

Приложение отсылает данные на сервер Google Analytics и там данные хранятся и обрабатываются. Вы через свою учётную запись на сервере можете просматривать полученную информацию.

Допустим, у вас уже есть популярное приложение, и вы хотите добавить сбор статистики, чтобы проанализировать данные. Для начала вам нужно получить аккаунт разработчика для получения доступа к данным. Страница регистрации (бесплатно) - http://www.google.com/analytics.

Допустим, вы вошли под своей учётной записью. Нажимаем на кнопку Администратор. Далее нажимаете на кнопку +Новый аккаунт. Появится страница выбора - Веб-сайт или Приложение. Для нашего случая выбираем второй вариант.

Введите имя приложения, чтобы не запутаться, если у вас будет много приложений. Хотя в принципе можно использовать один идентификатор для нескольких приложений.

Далее заполняете небольшую форму - нужно выбрать отрасль и часовой пояс.

Задайте имя для нового аккаунта и нажмите на кнопку Получить идентификатор отслеживания.

Вас попросят принять лицензионное соглашение. Соглашайтесь! У вас нет другого выхода.

Вам дадут идентификатор отслеживания вида UA-ХХХХХХХХ-Х. Запомните его. На странице вы также увидите кнопку для загрузки SDK. Возможны два варианта: для Android и для iOS. Кроме того есть вариан SDK с интегрированным AdMob.

Позже вы всегда можете вернуться на эту страницу - Администратор/Отслеживание информации.

Скачайте архив Google Analytics SDK for Android с страницы загрузки. Распакуйте архив и сохраните файл libGoogleAnalyticsV2.jar в папке /libs вашего проекта.

Добавьте библиотеку Google Analytics SDK for Android в ваш проект. Выберите Project | Java Build Path | Libraries. Щёлкните кнопку Add JARs… и выберите jar-файл из папки /libs.

Так как статистика отсылается через интернет, то необходимо прописать разрешения:


<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

Осталось добавить код отслеживания в приложение.

Добавьте методы объекта EasyTracker в методы onStart() и onStop() каждой вашей активности:


EasyTracker.getInstance().setContext(this);

  @Override
  public void onStart() {
    super.onStart();
    ... // The rest of your onStart() code.
    EasyTracker.getInstance().activityStart(this);
  }

  @Override
  public void onStop() {
    super.onStop();
    ... // The rest of your onStop() code.
    EasyTracker.getInstance().activityStop(this);
  }
}

Создаём новый xml-файл analytics.xml в папке res/values:


<?xml version="1.0" encoding="utf-8" ?>

<resources xmlns:tools="https://schemas.android.com/tools"
tools:ignore="TypographyDashes">
  <!--Замените на ваш идентификатор-->
  <string name="ga_trackingId">UA-XXXX-Y</string>

  <!--Enable automatic activity tracking-->
  <bool name="ga_autoActivityTracking">true</bool>

  <!--Enable automatic exception tracking-->
  <bool name="ga_reportUncaughtExceptions">true</bool>
</resources>

Чтобы Eclipse не ругался на использование символов "-" в идентификаторе, в тег resources был добавлен дополнительный атрибут tools:ignore. Это единственный правильный способ, заменять символ другим способом не нужно.

Этого минимума достаточно для начала работы с Google Analytics.

Примеры с исходниками можно найти на странице документации https://developers.google.com/analytics/devguides/collection/android/v2/advanced?hl=ru

Коротко о различных методах.

Отслеживаем активность через метод trackPageView() с указанием имени класса активности.


tracker.trackPageView("/Splash-Screen");

Отслеживаем события через метод trackEvent().


tracker.trackEvent("Clicks", "Button", "Easy", 0);

У метода четыре параметра:

  • category (обязательный) – строка, определюящая категорию события: щелчки, жесты, голосовые команды, а также такие события как play, pause, fast forward и т.д.
  • action (обязательный) – строка, определюящая действие события, например, мы сообщаем, что за нажатия отвечает кнопка.
  • label (необязательынй)– строка, связанная с событием. Если кнопок несколько, то позволит их различать по данному параметру.
  • value (необязательынй) – целое число Integer, задающее значение, связанное с событием. Помогает отслеживать количество элементов, котоыре завязаны на щелчке кнопок, например, число купленных вещей через кнопку Купить.
