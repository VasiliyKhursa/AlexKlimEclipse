Индикация загрузки страницы в WebView

Ровно месяц назад мы знакомились с элементом WebView, когда создавали свой браузер. Настало время освежить свои знания, а также научиться новым приёмам.

Мы научимся выводить изображения в WebView из сети, из приложения Галерея, а также из ресурсов.

Создадим новый проект и сразу пропишем разрешение на работу с интернетом.


<uses-permission android:name="android.permission.INTERNET" />

Создадим простую разметку из LinearLayout и WebView. К ним ещё добавим несколько кнопок для загрузки изображения из разных мест:


<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="fill_parent"
    android:layout_height="fill_parent"
    android:orientation="vertical" >

    <WebView
        android:id="@+id/picview"
        android:layout_width="fill_parent"
        android:layout_height="0dp"
        android:layout_weight="1" />

    <LinearLayout
        xmlns:android="http://schemas.android.com/apk/res/android"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:orientation="horizontal"
        android:padding="3dp" >

        <Button
            android:id="@+id/gallery_button"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
			android:onClick="onClick"
            android:text="@string/pick" />

        <Button
            android:id="@+id/web_button"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
			android:onClick="onClick"
            android:text="@string/load" />

        <Button
            android:id="@+id/local_button"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
			android:onClick="onClick"
            android:text="@string/local" />
    </LinearLayout>

</LinearLayout>

Определим несколько строковых ресурсов:


<string name="pick">Галерея</string>
<string name="load">Веб</string>
<string name="local">Ресурсы</string>

Подготовительные работы закончены. Можно приступать к написанию кода.

Загружаем из Галереи

Начнём с Галереи. Добавим новую переменную в классе:


private final int IMG_PICK = 1;

Напишем обработчик нажатия первой кнопки для выбора рисунка из Галереи:


case R.id.gallery_button:
	Intent pickIntent = new Intent();
	pickIntent.setType("image/*");
	pickIntent.setAction(Intent.ACTION_GET_CONTENT);
	// мы будем обрабатывать возвращенное значение в onActivityResult
	startActivityForResult(
			Intent.createChooser(pickIntent, "Выберите картинку"),
			IMG_PICK);
	break;

Данный код запускает намерение, которое ищет активность, работающую с изображениями. По умолчанию, на всех стандартных устройствах есть системное приложение Галерея, позволяющее рассматривать картинки, а также могут быть установлены другие приложения самим пользователем с схожим функционалом. Если таких программ на устройстве несколько, то появится диалоговое окно для выбора программы, если имеется только одна программа, то она запустится сразу (на эмуляторе обычно это Галерея).

Когда пользователь выбрал какую-то картинку (обычно это фотография котика), то нужно как-то узнать его выбор. Создадим метод onActivityResult(), который позволит нам выяснить выбор пользователя.

Если пользователь действительно что-то выбрал, то возвращается значение RESULT_OK. Далее проверяем наличие значения IMG_PICK, которое мы использовали при запуске Галереи. Если это значение вернулось, значит оно пришло от нужного приложения и мы можем продолжить обработку данных через метод getData(). Через объект Cursor извлекаем данные и получаем путь к выбранному файлу, который сразу же загружаем в WebView через метод loadUrl().


protected void onActivityResult(int requestCode, int resultCode, Intent data) {
	if (resultCode == RESULT_OK) {
		if (requestCode == IMG_PICK) {
			Uri pickedUri = data.getData();

			String imagePath = "";
			String[] imgData = { MediaStore.Images.Media.DATA };
			// Устаревший способ
			// Cursor imgCursor = managedQuery(pickedUri, imgData, null,
			// null, null);
			// Новый способ
			Cursor imgCursor = getApplicationContext().getContentResolver()
					.query(pickedUri, imgData, null, null, null);
			if (imgCursor != null) {
				int index = imgCursor
						.getColumnIndexOrThrow(MediaStore.Images.Media.DATA);
				imgCursor.moveToFirst();
				imagePath = imgCursor.getString(index);
			} else
				imagePath = pickedUri.getPath();
			picView.loadUrl("file:///" + imagePath);
		}
	}
}

Запустите проект и проверьте работоспособность кода.

Загружаем из интернета

Загрузить картинку из интернета очень просто, достаточно указать его адрес:


case R.id.web_button:
	picView.loadUrl("http://developer.alexanderklimov.ru/android/images/updowncat.jpg");
	break;

Этот способ нам знаком, останавливаться на нём не будем. Единственное, о чём нужно помнить - проверить наличие интернета и помнить, что при больших изображениях картинка будет грузиться достаточно долго и нужно предусмотреть индикатор прогресса.

Загружаем из ресурсов

Третий возможный способ загрузки - из внутренних ресурсов приложения. Это могут быть графические файлы из папки res/drawable или из папки res/assets. Рассмотрим вариант с последним случаем.


case R.id.local_button:
	picView.loadUrl("file:///android_asset/cat.jpg");
	break;

Если в папке res/assets хранится html-файл с картинками, то для тега img в этом файле прописывать путь к изображению нужно таким же образом:


<html>
<head>
</head>
<body>
<img src="file:///android_asset/cat.jpg"/>
</body>
</html>

После чего можно загрузить файл:


picView.loadUrl("file:///android_asset/page_with_image.htm");

Подгружаем свои шрифты

Также из ресурсов можно загрузить свой шрифт. Этот способ знаком веб-мастерам. Размещаем нужный шрифт в папке assets и прописываем ссылку на него в стилях:


<html>
<head>
<style type="text/css">
@font-face {
font-family: MyCustomFont;
src: url("file:///android_asset/fonts/Roboto.ttf")
}

body {
font-family: MyCustomFont;
font-size: 16pt;
color: black;
}
</style>
</head>
<body>
<p>Здесь ваш текст.</p>
</body>
</html>

Как загрузить страницу, мы уже знаем.

Дополнительная настройка WebView

Напомню, что у WebView есть различные настройки, которые могут быть полезны для просмотра картинки, в частности зум и прокрутка. Добавьте дополнительные строчки кода в методе onCreate():


picView.getSettings().setBuiltInZoomControls(true);
picView.getSettings().setUseWideViewPort(true);

Google Feed API

Возможность использования локальных страниц вполне удобна, если в ней использовать API, основанные на Javascript. Например, Google Feed API позволяет загружать данные Atom, RSS или Media RSS, используя только JavaScript. Вам остаётся подготовить локальную страницу с нужным сценарием и вывести результат в WebView.

Создадим в папке assets файл mypage.hmtl:


<html>
<meta charset=utf-8>
<head>
<script type="text/javascript" src="https://www.google.com/jsapi"></script>
<script type="text/javascript">

google.load("feeds", "1");

function initialize() {
var feed = new google.feeds.Feed("http://feeds.feedburner.com/alexanderklimov/VJcl");
feed.setNumEntries(10);
feed.load(function(result) {
  if (!result.error) {
    var container = document.getElementById("feed");
    for (var i = 0; i < result.feed.entries.length; i++) {
      var entry = result.feed.entries[i];
      var div = document.createElement("div");

      var A = document.createElement("A");
      A.setAttribute("href",entry.link);
      A.appendChild(document.createTextNode(entry.title));
      div.appendChild(A);

      container.appendChild(div);
    }
  }
});
}

google.setOnLoadCallback(initialize);

</script>
</head>
<body>
<div id="feed"></div>
</body>
</html>

Подключим страницу в коде активности:


@Override
public void onCreate(Bundle savedInstanceState) {
	super.onCreate(savedInstanceState);
	setContentView(R.layout.activity_test);
	
	WebView myWebView = (WebView)findViewById(R.id.web);
      
	myWebView.getSettings().setJavaScriptEnabled(true);
	myWebView.loadUrl("file:///android_asset/mypage.html");

}

В данном примере выводится десять последних записей из моей RSS-ленты. В эмуляторе пример вёл себя немного странно, выводя белый экран. Однако при щелчке на экране ссылки появлялись. Имейте виду. На реальном устройстве работало нормально.

Также вы можете подключить другие сценарии типа jQuery Mobile и сделать вывод результатов ещё лучше.

Индикация загрузки страницы в WebView

Нашёл здесь. Может пригодится.

При использовании WebView может появится необходимость отображать какое-нибудь диалоговое окно «Please wait», пока грузится страница. Хорошо, при старте загрузки страницы мы можем показать его, а вот когда его скрывать? Для этого необходимо установить свой WebViewClient и реализовать метод onPageFinished(). Да, это хорошо, но есть одно небольшое но. Данный метод вызовится как раз после того, как страница была полностью загружена, но если страница «тяжелая» – она может быть еще не отрендерина и, то есть, не показана пользователю. А рендеринг может занимать довольно длительное время, порой до нескольких секунд! В итоге если сделать примерно так:


public class WebViewClientTest extends Activity {

    /** Called when the activity is first created. */
    @Override
    public void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.main);
        final WebView webview = (WebView) findViewById(R.id.webview);
     
        webview.setWebViewClient(new WebViewClient() {
            @Override
            public void onPageFinished(WebView view, String url) {
                super.onPageFinished(webview, url);
                // прячем диалог please wait здесь
            }
        });
     
        // здесь выводим окно please wait
        webview.loadUrl("http://some URL");
    }
}

Если some URL будет довольно «тяжелый», то получим следующее: началась загрузка страницы, показывается диалог, потом диалог скрывается и мы видим пустую страницу и ожидаем, пока отрендерится и покажется что-то.

Как вариант, мы можем настроить для WebView слушатель PictureListener и прятать диалог в методе onNewPicture():


webView.setPictureListener(new PictureListener() {
    public void onNewPicture(WebView view, Picture picture) {
        // прячем диалог please wait здесь
    }
});

В итоге получим, что диалог исчезнет не после того, как страница загрузится, а после того, как она отобразится на экране.

Стоит отметить, что в Android 4 этот способ считается устаревшим. Пока не знаю, что является альтернативой.
