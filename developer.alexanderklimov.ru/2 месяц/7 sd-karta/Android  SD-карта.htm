Другие примеры

Кроме встроенной памяти, устройства могут иметь слот для подключения дополнительной флеш-памяти, обычно в виде SD-карты. Дополнительная память как правило больше встроенной, и на ней удобно хранить музыку, видео, сложные приложения навигации и т.д.

При записи на внешнюю карту необходим получить разрешение WRITE_EXTERNAL_STORAGE в манифесте приложения.

Например, для проигрывания видео с карты можно использовать код:


video.setVideoPath("/sdcard/sample.3gp"); // класс Video
video.start();

Получить путь к SD-карте

В большинстве случаев, путь к карте будет /sdcard и многие программисты жёстко прописывают данный путь в коде, но на самом деле это неправильный подход. Производители могут установить своё имя для карточки, поэтому для получения пути нужно использовать метод Environment.getExternalStorageDirectory().

Не совсем правильно воспринимать название функции, как доступ к внешней папке на SD, многие устройства имеют встроенную карточку памяти.

С этой функцией иногда нужно быть осторожной. Например, на устройстве Samsung Galaxy S3 метод возвращал /storage/sdcard0, но сама карточка была прописана как /storage/extSdCard. Чтобы избежать проблемы, приходилось вызывать метод Environment.getExternalStorageDirectory().getParent() и уже от этого плясать дальше (подробнее описано тут).

Если ваша программа хранит большие собственные данные, которые не должны быть доступны другим приложениям, и вам необходимо использовать SD-карту вместо встроенной памяти, то используйте специальный метод Context.getExternalFilesDir() для получения доступа к папке внешней файловой системы, где она может разместить файлы для постоянного хранения. Android удалит эти файлы, когда приложение будет деинсталлировано. В этом случае у пользователя будет порядок с папками и файлами, а не мешанина. В этом случае вам не требуется разрешение на запись.

Также есть метод getExternalStoragePublicDirectory(), который появился в API 8. Предназначен для сохранения файлов определённого типа для общего доступа. Например, картинке размещаются в папке /SD/Pictures (условно). Пример размещения картинки (из документации):


void createExternalStoragePublicPicture() {
    // Create a path where we will place our picture in the user's
    // public pictures directory.  Note that you should be careful about
    // what you place here, since the user often manages these files.  For
    // pictures and other media owned by the application, consider
    // Context.getExternalMediaDir().
    File path = Environment.getExternalStoragePublicDirectory(
            Environment.DIRECTORY_PICTURES);
    File file = new File(path, "DemoPicture.jpg");

    try {
        // Make sure the Pictures directory exists.
        path.mkdirs();

        // Very simple code to copy a picture from the application's
        // resource into the external file.  Note that this code does
        // no error checking, and assumes the picture is small (does not
        // try to copy it in chunks).  Note that if external storage is
        // not currently mounted this will silently fail.
        InputStream is = getResources().openRawResource(R.drawable.balloons);
        OutputStream os = new FileOutputStream(file);
        byte[] data = new byte[is.available()];
        is.read(data);
        os.write(data);
        is.close();
        os.close();

        // Tell the media scanner about the new file so that it is
        // immediately available to the user.
        MediaScannerConnection.scanFile(this,
                new String[] { file.toString() }, null,
                new MediaScannerConnection.OnScanCompletedListener() {
            public void onScanCompleted(String path, Uri uri) {
                Log.i("ExternalStorage", "Scanned " + path + ":");
                Log.i("ExternalStorage", "-> uri=" + uri);
            }
        });
    } catch (IOException e) {
        // Unable to create file, likely because external storage is
        // not currently mounted.
        Log.w("ExternalStorage", "Error writing " + file, e);
    }
}

void deleteExternalStoragePublicPicture() {
    // Create a path where we will place our picture in the user's
    // public pictures directory and delete the file.  If external
    // storage is not currently mounted this will fail.
    File path = Environment.getExternalStoragePublicDirectory(
            Environment.DIRECTORY_PICTURES);
    File file = new File(path, "DemoPicture.jpg");
    file.delete();
}

boolean hasExternalStoragePublicPicture() {
    // Create a path where we will place our picture in the user's
    // public pictures directory and check if the file exists.  If
    // external storage is not currently mounted this will think the
    // picture doesn't exist.
    File path = Environment.getExternalStoragePublicDirectory(
            Environment.DIRECTORY_PICTURES);
    File file = new File(path, "DemoPicture.jpg");
    return file.exists();
}

При использовании метода getExternalStoragePublicDirectory() картинки остаются на месте при удалении программы.

В примере использовалась константа DIRECTORY_PICTURES, кроме неё есть и другие константы:

  • DIRECTORY_ALARMS
  • DIRECTORY_DCIM
  • DIRECTORY_DOWNLOADS
  • DIRECTORY_MOVIES
  • DIRECTORY_MUSIC
  • DIRECTORY_NOTIFICATIONS
  • DIRECTORY_PICTURES
  • DIRECTORY_PODCASTS
  • DIRECTORY_RINGTONES

Программное создание папки на SD-карте

Для создания папки можно использовать стандартный Java-код с использованием метода mkdir() класса java.io.File (не забываем про разрешение android.permission.WRITE_EXTERNAL_STORAGE).


String newFolder = "/CatAlbum";
// получаем путь к SD-карте
String extStorageDirectory = Environment.getExternalStorageDirectory().toString();
File myNewFolder = new File(extStorageDirectory + newFolder);
myNewFolder.mkdir();

Проверить наличие SD-карты на устройстве

Чтобы убедиться, что SD-карта имеется на устройстве, я написал такую функцию проверки (Виктор Аленьков предложил позже укороченный вариант):


// укороченный вариант
private boolean isSDCardReady() {
	return (Environment.getExternalStorageState().equals(Environment.MEDIA_MOUNTED));
}

// первоначальный вариант
private boolean isSDCardReady() {
	if (Environment.getExternalStorageState().equals(
			Environment.MEDIA_MOUNTED)) {
		// SD-карта вставлена
		return true;
	} else {
		// SD-карты нет
		return false;
	}
}

Определить, сколько свободного места осталось на SD-карте

Если вам необходимо узнать, сколько свободного места осталось на SD-карте, то воспользуйтесь следующим кодом:


public static final float BYTE_IN_MB = 1024f*1024f;

private static int getSpaceForFile(String path) {

	StatFs statfs = new StatFs(path);
	final int blocks = statfs.getAvailableBlocks();
	final int blockSize = statfs.getBlockSize();
	final int totalBytes = blocks * blockSize;
	// переводим в мегабайты
	final int freespace = (int) (totalBytes / BYTE_IN_MB);
	
	return freespace;
}

public static int getExternalFreeSpace() {

	final String path = Environment.getExternalStorageDirectory().getPath();
	final int freespace = getSpaceForFile(path);

	return freespace;
}

public static int getInternalFreeSpace() {

	final String path = Environment.getDataDirectory().getPath();
	final int freespace = getSpaceForFile(path);

	return freespace;
}

Метод getExternalFreeSpace() использует вспомогательный метод getSpaceForFile(), который можно использовать и для получения информации о свободном месте на внутренней карточке памяти.

Полная информация о SD-карте

Объединив полученные данные, можно написать приложение, которое определяет доступность внешней карточки и выводит информация о полном объёме хранилища и сколько осталось места (в байтах, килобайтах, мегабайтах). В разметку добавьте два TextView:


package ru.alexanderklimov.sdcard;

import ...

public class TestActivity extends Activity {
	// the statistics of the SD card
	private StatFs stats;
	// the state of the external storage
	private String externalStorageState;

	// the total size of the SD card
	private double totalSize;
	// the available free space
	private double freeSpace;

	// a String to store the SD card information
	private String outputInfo;
	// a TextView to output the SD card state
	private TextView tv_state;
	// a TextView to output the SD card information
	private TextView tv_info;

	// set the number format output
	private NumberFormat numberFormat;

	@Override
	public void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);
		setContentView(R.layout.activity_test);

		// initialize the Text Views with the data at the main.xml file
		tv_state = (TextView) findViewById(R.id.tv_state);
		tv_info = (TextView) findViewById(R.id.tv_info);

		// get external storage (SD card) state
		externalStorageState = Environment.getExternalStorageState();

		// checks if the SD card is attached to the Android device
		if (externalStorageState.equals(Environment.MEDIA_MOUNTED)
				|| externalStorageState.equals(Environment.MEDIA_UNMOUNTED)
				|| externalStorageState
						.equals(Environment.MEDIA_MOUNTED_READ_ONLY)) {
			// obtain the stats from the root of the SD card.
			stats = new StatFs(Environment.getExternalStorageDirectory()
					.getPath());

			// Add 'Total Size' to the output string:
			outputInfo = "\nTotal Size:\n";

			// total usable size
			totalSize = stats.getBlockCount() * stats.getBlockSize();

			// initialize the NumberFormat object
			numberFormat = NumberFormat.getInstance();
			// disable grouping
			numberFormat.setGroupingUsed(false);
			// display numbers with two decimal places
			numberFormat.setMaximumFractionDigits(2);

			// Output the SD card's total size in gigabytes, megabytes,
			// kilobytes and bytes
			outputInfo += "Size in gigabytes: "
					+ numberFormat.format((totalSize / (double) 1073741824))
					+ " GB \n" + "Size in megabytes: "
					+ numberFormat.format((totalSize / (double) 1048576))
					+ " MB \n" + "Size in kilobytes: "
					+ numberFormat.format((totalSize / (double) 1024))
					+ " KB \n" + "Size in bytes: "
					+ numberFormat.format(totalSize) + " B \n";

			// Add 'Remaining Space' to the output string:
			outputInfo += "\nRemaining Space:\n";

			// available free space
			freeSpace = stats.getAvailableBlocks() * stats.getBlockSize();

			// Output the SD card's available free space in gigabytes,
			// megabytes, kilobytes and bytes
			outputInfo += "Size in gigabytes: "
					+ numberFormat.format((freeSpace / (double) 1073741824))
					+ " GB \n" + "Size in megabytes: "
					+ numberFormat.format((freeSpace / (double) 1048576))
					+ " MB \n" + "Size in kilobytes: "
					+ numberFormat.format((freeSpace / (double) 1024))
					+ " KB \n" + "Size in bytes: "
					+ numberFormat.format(freeSpace) + " B \n";

			// output the SD card state
			tv_state.setTextColor(Color.GREEN);
			tv_state.setText("SD card found! SD card is "
					+ externalStorageState + ".");

			// output the SD card info
			tv_info.setText(outputInfo);
		} else // external storage not found
		{
			// output the SD card state
			tv_state.setTextColor(Color.RED);
			tv_state.setText("SD card not found! SD card state is \""
					+ externalStorageState + "\".");
		}
	}
}

Установка приложения на SD-карту

Начиная с версии Android 2.2, у разработчика появилась возможность задать установку программы не только во внутреннюю память телефона, но и на SD-карту. Просто добавьте атрибут android:installLocation = "" в теге manifest. Можно использовать два значения: auto и preferExternal.

Значение auto позволяет системе самой решать, куда установить программу, параметр preferExternal запрашивает у системы установку (но не гарантирует). У пользователя появляется возможность перемещать установленное приложение между внутренней и внешней памятью с помощью установки приложения.

Есть еще параметр internalOnly, который указывает на установку во внутреннюю память, но её можно не использовать, так как отсутствие атрибута приводит к такому же результату.

Следует осторожно использовать возможность установки приложения на SD-карту, ведь её можно в любой момент вытащить из телефона. Google не рекомендует использовать установку на SD-карту, если приложение имеет:

  • управление учетными записями
  • таймеры
  • интерактивные папки
  • сервисы
  • адаптеры синхронизации
  • виджеты на рабочем столе

Загрузка изображения с карточки в ImageView

Если вам известен путь к изображению, которое хранится на SD-карте, то можно использовать следующий код:


TextView textJpgName = (TextView)findViewById(R.id.textviewjpgname);
ImageView jpgView = (ImageView)findViewById(R.id.jpgview);

String myJpgPath = "/sdcard/DSC_3524.JPG"; //путь к картинке. У вас будет свой

textJpgName.setText(myJpgPath); // выводим путь в текстовом поле

BitmapFactory.Options options = new BitmapFactory.Options();
options.inSampleSize = 2;
Bitmap bm = BitmapFactory.decodeFile(myJpgPath, options);
jpgView.setImageBitmap(bm);

Здесь нужно обратить вниманию на одну особенность. При загрузке слишком большого изображения (особенно на устройствах с мощной камерой) вылезает ошибка. Поэтому рекомендуется уменьшить размер картинки при помощи свойства inSampleSize для экономии памяти.

Сохранение картинки из ImageView на SD-карту

Рассмотрим противоположную задачу - сохранение картинки из ImageView на SD-карту. Кстати, этот способ можно использовать для передачи картинок из одной активности в другую. Сохранив изображение на внешнюю память, вам достаточно передать второй активности строку с адресом картинки.


bmImage = (ImageView) findViewById(R.id.image1);

public void onClick(View view) {
	bmImage.setDrawingCacheEnabled(true);
	Bitmap bitmap = bmImage.getDrawingCache();

	File picDir = Environment
			.getExternalStoragePublicDirectory(Environment.DIRECTORY_PICTURES);
	File picFile = new File(picDir + "/" + "cat.png");

	try {
		picFile.createNewFile();
		FileOutputStream picOut = new FileOutputStream(picFile);
		boolean worked = bitmap.compress(CompressFormat.PNG, 100,
				picOut);
		if (worked) {
			Toast.makeText(
					getApplicationContext(),
					"Image saved to your device Pictures "
							+ "directory!", Toast.LENGTH_SHORT).show();
		} else {
			Toast.makeText(getApplicationContext(),
					"Whoops! File not saved.", Toast.LENGTH_SHORT)
					.show();
		}
		picOut.close();
	} catch (FileNotFoundException e) {
		// TODO Auto-generated catch block
		e.printStackTrace();
		Toast.makeText(TestActivity.this, e.toString(), Toast.LENGTH_LONG)
				.show();
	} catch (IOException e) {
		// TODO Auto-generated catch block
		e.printStackTrace();
		Toast.makeText(TestActivity.this, e.toString(), Toast.LENGTH_LONG)
				.show();
	}
}
