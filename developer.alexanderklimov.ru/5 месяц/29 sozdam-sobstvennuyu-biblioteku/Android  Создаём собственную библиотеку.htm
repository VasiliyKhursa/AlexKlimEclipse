Создаём собственную библиотеку

Закрытая зона/5-й месяц

Рассмотрим создание собственной библиотеки. Вам уже не раз приходилось пользоваться чужими библиотеками. Пора научиться создавать собственные библиотеки, которые можно будет подключать к своим проектам или делиться ими с людьми.

Если расширением для приложений является apk-файл, то для библиотек - это jar-файл. JAR-файл нельзя распространять отдельно в Google Play, его можно использовать только в составе приложений.

Наверняка, вы замечали, что при создании нового проекта, флажок для создания библиотеки всегда оставался нетронутым. Давайте теперь попробуем его задействовать.

Создаём новый проект File | New | Android Application Project. В первом диалоговом окне заполняем поля как обычно - имя приложения, проекта, пакета. В следующем окне New Android Application будьте внимательнее.

Во-первых, снимите флажок у пункта Create custom launcher icon, так как в библиотеке не используется значок.

Во-вторых, снимите флажок у пункта Create Activity, так как мы не будем создавать активность для нашего примера.

Отметьте флажком пункт Mark this project as a library - ведь мы собираемся создавать библиотеку.

Структура проекта останется такой же, ничего нового вы не увидите. Но небольшое отличие всё-таки имеется. В папке src нет пакета и класса, как обычно мы привыкли наблюдать в стандартных проектах.

Создадим их вручную. Щёлкаем правой кнопкой мыши по папке src и выбираем команду New | Package. В диалоговом окне вводим нужное имя для пакета.

Далее нужно создать класс. Здесь вам должно быть понятно.

Пишем код для класса.

Далее есть два способа использования библиотеки. Так как мы пометили проект как библиотеку, то в обычном проекте можно сделать ссылку на созданную библиотеку. Возможно, вам приходилось подключать библиотеки подобным образом. Но мы сейчас рассмотрим другой вариант с использованием jar-файла.

Чтобы получить jar-файл, нужно выбрать в меню Project | Build Project и в папке bin на вашем компьютере появится файл catlibrary.jar. Если команда меню не доступна, то попробуйте сначала снять флажок Build Automaticaly.

Итак, у нас есть файл библиотеки. Попробуем его использовать. Создадим стандартный проект и скопируем jar-файл из Проводника в окно Eclipse в папку libs стандартного проекта. Eclipse должен автоматически подхватить библиотеку и корректно импортировать класс из библиотеки.

В методе onCreate() подключаем компонент:


package ru.alexanderklimov.test;

import ru.alexanderklimov.catlibrary.CatHouse;
import android.app.Activity;
import android.graphics.Color;

import android.os.Bundle;
import android.view.View;

public class TestActivity extends Activity {

	/** Called when the activity is first created. */
	@Override
	public void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);

		// setContentView(R.layout.activity_test);

		CatHouse cathouse = new CatHouse(this, 3, Color.BLUE, Color.rgb(255,
				215, 0));
		setContentView(cathouse);

	}
}

Согласен, не очень похоже на дом. Я немного слукавил. Мне было лень рисовать дом и я взял готовый код класса GameBoard из примера в одной книге, который на самом деле рисует игровое поле. Например, можно вывести на экран шахматную доску:


CatHouse cathouse = new CatHouse(this, 8, Color.BLACK, Color.WHITE);

или, если переименовать класс:


GameBoard board = new GameBoard(this, 8, Color.BLACK, Color.WHITE);
setContentView(board);

Таким образом, создав библиотеку для рисования игровой доски, вы можете поделиться наработкой с другими программистами, которые могут подключить её к своему проекту и написать различные игры - шахматы, шашки, уголки, поддавки и т.д. Также можно задать другие параметры для доски, чтобы получить игровое поле для других игр, например, го, рэндзю, реверси и т.д.

Имейте в виду, что мы создали библиотеку определённого типа, который подходит для создания нового компонента. Но мы не можем, например, использовать ресурсы из папки drawable и других папок. В таких случаях используется другой подход, который я не изучал.
