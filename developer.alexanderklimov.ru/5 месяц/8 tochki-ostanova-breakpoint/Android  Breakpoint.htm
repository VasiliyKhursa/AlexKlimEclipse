Точки останова Breakpoint

Рассмотрим вопрос отладки приложений в Eclipse (многие просили осветить данную тему).

Надеюсь, вы уже знаете, как пользоваться инструментом LogCat и классом android.util.Log. На сайте уже есть материал на данную тему: Отладка. Изучим другие приёмы, которые помогут лучше контролировать код программы.

Создадим простой проект, в котором будем складывать числа. Начнём с разметки:


<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="vertical" >

    <TextView
        android:id="@+id/textView1"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_gravity="center_horizontal"
        android:layout_marginTop="18dp"
        android:text="i = 0"
        android:textAppearance="?android:attr/textAppearanceLarge" />

    <Button
        android:id="@+id/button1"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_gravity="center_horizontal"
        android:text="Выполнить (i = i + 1)" />

    <Button
        android:id="@+id/button2"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:layout_gravity="center_horizontal"
        android:text="Выполнить (i = i + 3)" />

</LinearLayout>

У нас есть две кнопки и при щелчках будем прибавлять к переменной i значение 1 или 3. Напишем соответствующий код:


public class TestActivity extends Activity {
	int i = 0;

	@Override
	public void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);

		setContentView(R.layout.activity_test);

		final TextView txtView1 = (TextView) findViewById(R.id.textView1);

		// Первая кнопка (i = i + 1)
		final Button button1 = (Button) findViewById(R.id.button1);
		button1.setOnClickListener(new View.OnClickListener() {
			public void onClick(View v) {
				i = i + 1;
				Log.d("Шаг 1", "i = " + i);

				txtView1.setText("i = " + i); // выводим в TextView
			}
		});

		// Вторая кнопка (i = i + 3)
		final Button button2 = (Button) findViewById(R.id.button2);
		button2.setOnClickListener(new View.OnClickListener() {
			public void onClick(View v) {
				i = i + 3;
				Log.d("Шаг 2", "i = " + i);

				txtView1.setText("i = " + i); // выводим в TextView
			}
		});

	}
}

Пока всё знакомо, чем-то напоминает программу Счётчик ворон, которую вы писали в самом начале изучения Android. Результат сложения выводим в текстовую метку, а также в LogCat.

Теперь начинается магия. В редакторе кода Eclipse слева идёт тонкая полоска перед нумерацией строчек кода. Обычно мы видим там различные треугольники и предупреждающие значки. Если щелкнуть по ней правой кнопкой, то откроется контекстное меню, в котором есть команда Toggle Breakpoint (Ctrl+Shift+B), которая устанавливает точку останова. Установите курсор напротив строки final TextView txtView1 = (TextView) findViewById(R.id.textView1); и поставьте первую точку останова. Появится маленький зелёный кружочек. Аналогично поставьте вторую и третью точку останова у строчек txtView1.setText("i = " + i); и txtView1.setText("i = " + i);.

Далее в меню выбираем Run | Debug (F11). Появится диалоговое окно Confirm Perspective Switch:

Приложение запустится и остановится на первой точке останова. Сама строка кода будет подсвечена зелёным цветом.

Найдите вкладки Variables и Breakpoints.

Далее можете проходить пошагово по строкам кода. На вкладке Variables можете отслеживать значение переменной i в текущий момент.

Если во вкладке Variables щелкнуть правой кнопкой мыши и в контекстном меню выбрать пункт Watch, то откроется ещё одна вкладка Expressions.

Щелкните по строчке Add new expression и введите имя переменной i, чтобы сразу увидеть её значение. Это удобно, когда переменных очень много и искать обычной прокруткой тяжело.

Также на панели инструментов есть кнопки Resume для перехода к новой точке останова, а также кнопки Step Into, Step Over, Step Return и др.

Сам я не использую возможности отладки на полную катушку, так как не пишу серьёзных проектов. Но в компаниях использование подобных инструментов является обязательным условием.

Существуют и другие приёмы отладки приложений. Об этом в другой раз.
