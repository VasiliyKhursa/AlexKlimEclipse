Внедряем рекламу AdMob в своё приложение

Закрытая зона/5-й месяц

Метки: AdMob

Если вы написали популярную программу, которой с удовольствием пользуются люди, то вы можете монетизировать приложене, добавив в неё рекламу, чтобы отбить расходы.

Заходим на страницу Google Mobile Ads SDK (на русском). Изучаем материалы, статьи на русском и понятны, хотя встречаются небольшие погрешности. В статье использованы материалы с сайта, поэтому заглядывайте на сам сайт, чтобы быть в курсе последний изменений.

Скачиваем библиотеку (на данный момент версия 6.4.1). В состав архива входит библиотека, документация и дополнительные материалы. Также на странице загрузки можно скачать тестовый пример.

Прежде чем создавать проекты с рекламой, необходимо зарегистрироваться на сайте AdMob. После регистрации вам необходимо добавить своё приложение на сайт, чтобы получить свой код издателя, а также отслеживать статистику. На вкладке Sites & Apps выбирайте ссылку Add Site/App и на новой странице выбирайте значок с зелёным человечком (Android App). Заполнив необходимые поля, вводите капчу и завершаете процесс регистрации приложения.

Снова возвращаетесь на страницу со списком ваших зарегистрированных программ. Если подвести курсор мыши к названию программы, то появится кнопка Manage Settings. Нажав на неё, вы попадёте на страницу, на которой можно узнать свой идентификатор издателя (Publisher ID). Скопируйте его - он понадобится в вашем проекте. На этом подготовительная часть закончена.

Создаём собственный проект и добавляем в него библиотеку. Далее следует прописать активность в манифесте:


<activity android:name="com.google.ads.AdActivity"
        android:configChanges="keyboard|keyboardHidden|orientation|screenLayout|uiMode|screenSize|smallestScreenSize"/>

Там же прописываем необходимые разрешения:


<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>

Рекламный блок можно вставлять в активности программно или через XML. Рассмотрим пример программного добавления баннера.


package ru.alexanderklimov.test;

import android.app.Activity;
import android.os.Bundle;
import android.widget.LinearLayout;

import com.google.ads.AdRequest;
import com.google.ads.AdSize;
import com.google.ads.AdView;

public class TestActivity extends Activity {

	private AdView adView;

	/** Called when the activity is first created. */
	@Override
	public void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);

		setContentView(R.layout.activity_test);

		// Создание adView
		adView = new AdView(this, AdSize.BANNER, "a64f689413d4524");

		// Поиск в LinearLayout (предполагается, что был назначен
		// атрибут android:id="@+id/mainLayout"
		LinearLayout layout = (LinearLayout) findViewById(R.id.linearLayout);

		// Добавление adView
		layout.addView(adView);

		AdRequest adRequest = new AdRequest();
		adRequest.addTestDevice(AdRequest.TEST_EMULATOR); // Эмулятор

		// Инициирование общего запроса на загрузку вместе с объявлением
		adView.loadAd(adRequest);
	}

	@Override
	public void onDestroy() {
		adView.destroy();
		super.onDestroy();
	}
}

В строке кода adView = new AdView(this, AdSize.BANNER, "a64f689413d4524"); нужно заменить последний параметр на ваш идентификатор, о котором я говорил вам раньше. Так как проверять сначала мы будем на эмуляторе, то используйте константу AdRequest.TEST_EMULATOR при вызове метода addTestDevice(). Если всё сделали правильно, то при запуске проекта вы увидите баннер в своём приложении.

Размещение баннера через XML-разметку удобнее, так как вы сможете визуально расположить рекламный блок в нужном месте.


<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:ads="http://schemas.android.com/apk/lib/com.google.ads"
    android:layout_width="fill_parent"
    android:layout_height="fill_parent"
    android:orientation="vertical" >

    <com.google.ads.AdView
        android:id="@+id/adView"
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        ads:adSize="BANNER"
        ads:adUnitId="MY_AD_UNIT_ID"
        ads:loadAdOnCreate="true"
        ads:testDevices="TEST_EMULATOR, TEST_DEVICE_ID" />

</LinearLayout>

Заменить значение MY_AD_UNIT_ID своим идентификатором издателя AdMob. Чтобы получать тестовые объявления на свое устройство, необходимо добавить его идентификатор в атрибут ads:testDevices. Теперь код писать не обязательно, баннер загрузится автоматически.

Если вам нужен контроль над запросом (AdRequest), используемым для загрузки объявления в приложение, вы можете удалить строку ads:loadAdOnCreate="true" из XML-кода. Вместо этого можно получить доступ к AdView программно с помощью метода loadAd():


@Override
public void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.main);

    // Поиск AdView в качестве ресурса и загрузка запроса.
    AdView adView = (AdView)this.findViewById(R.id.adView);
    adView.loadAd(new AdRequest());
}

Константа BANNER в атрибуте ads:adSize отвечает за баннер размером 320 x 50. Это стандартный размер для телефонов и планшетов. Также можно использовать другие размеры для планшетов. Смотри полную таблицу размеров.

Размер (Ш x В) Описание Устройства Константа AdSize
320 x 50 Стандартный баннер Телефоны и планшетные ПК BANNER
300 x 250 Средний прямоугольник (IAB) Планшетные ПК IAB_MRECT
468 x 60 Полный баннер (IAB) Планшетные ПК IAB_BANNER
728 x 90 Полноразмерный баннер (IAB) Планшетные ПК IAB_LEADERBOARD
SDK запрашивает тот размер, который был задан при создании экземпляра AdView. Если на экране устройства недостаточно места для показа объявления, оно показано не будет.

AdRequest.addTestDevice | AdRequest.setTestDevices

Эти свойства можно использовать для указания устройства или набора (Set) устройств, которые будут получать тестовые объявления. При разработке следует задать значение этого свойства, чтобы избежать нерелевантных показов. Чтобы проверить правильность интеграции SDK, добавьте устройство для тестирования, запустите приложение и нажмите на тестовое объявление.


AdRequest request = new AdRequest();

request.addTestDevice(AdRequest.TEST_EMULATOR);
request.addTestDevice("E83D20734F72FB3108F104ABC0FFC738");    // Тестовый телефон T-Mobile G1

С помощью команды logcat можно просмотреть MD5-хеш идентификатора:

adRequest.addTestDevice("E83D20734F72FB3108F104ABC0FFC738");

Таргетинг

Можно также указывать информацию для географического и демографического таргетинга. Из соображений конфиденциальности пользователей используйте географические и демографические данные, только если они уже используются в приложении.


AdRequest request = new AdRequest();

request.setGender(AdRequest.Gender.FEMALE);
request.setLocation(location);
request.setBirthday("19850101");

com.google.ads.AdListener

При необходимости можно отслеживать события жизненного цикла объявления, например клики и неудачные запросы, путем реализации интерфейса com.google.ads.AdListener в объекте, передаваемом в AdView.setAdListener.


public interface AdListener {
    public void onReceiveAd(Ad ad);
    public void onFailedToReceiveAd(Ad ad, AdRequest.ErrorCode error);
    public void onPresentScreen(Ad ad);
    public void onDismissScreen(Ad ad);
    public void onLeaveApplication(Ad ad);
}

Этот интерфейс может быть реализован в Activity или любом другом объекте:


import com.google.ads.*;

public class BannerExample extends Activity implements AdListener {
}

Затем он передается в объект Ad:


adView.setAdListener(this);

public void onReceiveAd(Ad ad)

Вызывается при успешном выполнении AdView.loadAd.

public void onFailedToReceiveAd(Ad ad, AdRequest.ErrorCode error)

Вызывается при ошибке выполнения метода loadAd , которая обычно связана с сетевым сбоем, некорректной настройкой приложения или отсутствием подходящих объявлений. Такие события можно фиксировать в журналах для последующей отладки:


@Override
public void onFailedToReceiveAd(Ad ad, AdRequest.ErrorCode errorCode) {
  Log.d(MY_LOG_TAG, "failed to receive ad (" + errorCode + ")");
}

public void onPresentScreen(Ad ad)

Вызывается, когда Activity создается поверх приложения, вследствие чего пользователю в ответ на нажатие объявления показывается полноэкранная реклама.

public void onDismissScreen(Ad ad)

Вызывается, когда полноэкранный объект Activity, представленный в onPresentScreen, закрывается и управление возвращается приложению.

public void onLeaveApplication(Ad ad)

Вызывается, когда при нажатии на объявление (Ad) запускается новое приложение.
