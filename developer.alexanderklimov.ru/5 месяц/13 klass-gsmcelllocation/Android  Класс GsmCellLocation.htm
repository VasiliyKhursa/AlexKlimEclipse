Класс GsmCellLocation (Черновик)

Местоположение можно определять не только через спутники по GPS-приёмнику, но и через идентификатор текущей вышки оператора – Cell ID и Lac (область). Android предоставляет API для получения этих значений - класс android.telephony.gsm.GsmCellLocation.

Телефон пользователя всегда на связи, а значит подключён к сотовой сети, т.е. к определённой вышке. Зная номер вышки, мы можем узнать приблизительное местоположение пользователя. Следует учитывать, что область приёма сигнала вышкой может варьироваться от нескольких сотен метров до нескольких километров. Тем не менее и этого бывает достаточно для многих задач.

У класса есть методы getCid() и getLac(), которые возвращает целые числа. Зная эти числа, можно подключиться к какой-нибудь базе, которая хранит информацию об этих номерах. Например, я вбил значения со своего телефона на сайте http://map.online-gsm.ru/loc/ и получил своё местонахождение.

Для справки - MCC для России - 250, MNC для Билайна - 99. Для ваших операторов ищите информацию самостоятельно.

Ниже приводится только простенький код, чтобы не забыть про данную тему. Буду благодарен, если вы найдёте хороший туториал на эту тему, чтобы написать полноценную статью.


public int getCID(Context ctx) {
	try {
		TelephonyManager tm = (TelephonyManager) ctx
				.getSystemService(Context.TELEPHONY_SERVICE);
		GsmCellLocation location = (GsmCellLocation) tm.getCellLocation();

		int locationCellid = location.getCid();
		lac = location.getLac();
		
		int cellId = -1; // set to unknown location by default
		
		if (locationCellid > 0) { // known location
			cellId = locationCellid; // & 0xffff; // get only valuable bytes
			
		}
		
		return cellId;
	} catch (Exception ignored) {
	}

	return -1;
}

public void onClick(View v) {
	int cid = getCID(getApplicationContext());
	TextView tvInfo = (TextView)findViewById(R.id.textView1);
	TextView tvLac = (TextView)findViewById(R.id.tvLac);
	tvInfo.setText("cid: " + cid);
	tvLac.setText("LAC: " + lac);
	
}

Разрешение в манифесте


<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/ >

Значения CellID и LAC должны находиться в диапазоне от 0 до 0xffff. В противном случае считается, что показания нельзя обработать.

Можно передать запрос сервису Яндекса - http://mobile.maps.yandex.net/cellid_location/?&cellid=%d&operatorid=%d&countrycode=%d&lac=%d, подставив вместо %d нужные значения. Вам вернется ответ в виде XML:

<location source="FoundByCellid">
<coordinates latitude="55.8735046" longitude="37.4952126" nlatitude="55.8788195" nlongitude="37.5046862"/>

Существует проекты на Западе с такой же функциональностью.
