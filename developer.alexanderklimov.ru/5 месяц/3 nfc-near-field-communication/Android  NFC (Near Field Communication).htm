NFC (Near Field Communication)

Метки: NFC, NfcAdapter, MifareUltralight

Общая информация
Проверяем наличие NFC-адаптера
Считываем метку

Многие новые модели устройств на Android стали поддерживать модную технологию NFC (Near Field Communication). Полноценная поддержка NFC впервые появилась с API 10 (Android 2.3.3).

Near Field Communication («коммуникация ближнего поля») — технология беспроводной высокочастотной связи малого радиуса действия, которая даёт возможность обмена данными между устройствами, находящимися на расстоянии около 10 сантиметров (около 4-х дюймов). Устройство NFC может поддерживать связь и с существующими смарткартами и считывателями стандарта ISO 14443, и с другими устройствами NFC, и таким образом совместимо с существующей инфраструктурой бесконтактных карт, уже использующейся в общественном транспорте и платежных системах. NFC нацелена прежде всего на использование в мобильных телефонах. (Википедия).

Общая информация

Для работы с NFC следует прописать специальное разрешение в манифесте:


<uses-permission android:name="android.permission.NFC" />

Если приложение использует NFC и вы хотите, чтобы у пользователей старых устройств не было возможности скачивать ваше приложение из Google Play, то установите необходимый фильтр в том же манифесте:


<uses-feature android:name="android.hardware.nfc" android:required="true" />

С учётом того, что нормальная поддержка NFC появилась в API 10, следует устанавливать минимальный уровень в манифесте:


<uses-sdk android:minSdkVersion="10"/>

Для работы с NFC есть пять классов в пакете android.nfc:

  • NdefMessage - сообщения в стандарте NDEF
  • NdefRecord - запись в стандарте NDEF
  • NfcAdapter - адаптер NFC на устройстве
  • NfcManager - менеджер NFC
  • Tag - обработчик тегов NFC

Также имеется пакет android.nfc.tech для работы с тегами.

Три основных намерения для работы с NFC:

  • android.nfc.action.NDEF_DISCOVERED
  • android.nfc.ation.TECH_DISCOVERED
  • android.nfc.action.TAG_DISCOVERED

Проверяем наличие NFC-адаптера

Чтобы убедиться, что на устройстве есть NFC-адаптер, можно устроить небольшую проверку:


private NfcAdapter mAdapter;

mAdapter = NfcAdapter.getDefaultAdapter(this);

if (mAdapter == null) {
	Toast.makeText(this, "Sorry, NFC is not available on this device",
			Toast.LENGTH_SHORT).show();
} else {
	Toast.makeText(this, mAdapter.toString(), Toast.LENGTH_SHORT)
			.show();
}

В блоке else выводится имя адаптера в виде имени и набора цифр. В реальном приложении такой надобности нет, я привел просто для демонстрации.

Считываем метку

Существует несколько протоколов, используемых в NFC. У меня под рукой оказалась только карточка проезда на московском метро, поэтому я сосредоточился на изучении вопроса считывания метки с этой карточки.

В манифесте необходимо прописать фильтры для работы с NFC.


<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="ru.alexanderklimov.nfctag"
    android:versionCode="1"
    android:versionName="1.0" >

    <uses-sdk
        android:minSdkVersion="10"
        android:targetSdkVersion="17" />

    <uses-permission android:name="android.permission.NFC" />

    <application
        android:allowBackup="true"
        android:icon="@drawable/ic_launcher"
        android:label="@string/app_name"
        android:theme="@style/AppTheme" >
        <activity
            android:name="ru.alexanderklimov.nfctag.MainActivity"
            android:label="@string/app_name"
            android:launchMode="singleTop" >
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />

                <category android:name="android.intent.category.LAUNCHER" />

                <action android:name="android.nfc.action.TECH_DISCOVERED" />
            </intent-filter>

            <meta-data
                android:name="android.nfc.action.TECH_DISCOVERED"
                android:resource="@xml/filter_nfc" />
        </activity>
    </application>

</manifest>

Обратите внимание на <action android:name="android.nfc.action.TECH_DISCOVERED" />, на также на секцию meta-data. Существуют и другие параметры, о которых можно прочитать в документации. Я оставил только то, что пригодится для работы с картой метро.

В мета-данных встречается ссылка на файл filter_nfc.xml, который должен находится в папке res/xml:


<?xml version="1.0" encoding="utf-8"?>
<resources xmlns:xliff="urn:oasis:names:tc:xliff:document:1.2">
    <tech-list>
        <tech>android.nfc.tech.NfcF</tech>
    </tech-list>

    <!-- capture all MIFARE Classics with NDEF payloads -->
    <tech-list>
        <tech>android.nfc.tech.NfcA</tech>
        <tech>android.nfc.tech.MifareClassic</tech>
        <tech>android.nfc.tech.Ndef</tech>
        <tech>android.nfc.tech.MifareUltralight</tech>
    </tech-list>
</resources>

В реальности для карточки метро нужна только последняя строчка <tech>android.nfc.tech.MifareUltralight</tech>, остальные я просто копировал с документации при изучении.

Создадим разметку для экрана, состоящую из текстовой метки:


<?xml version="1.0" encoding="utf-8"?>
<TextView xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/text"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:gravity="center_vertical|center_horizontal"
    android:textAppearance="?android:attr/textAppearanceMedium" />

Теперь код для активности:


package ru.alexanderklimov.nfctag;

import java.io.IOException;
import java.nio.charset.Charset;

import android.app.Activity;
import android.app.PendingIntent;
import android.content.Intent;
import android.content.IntentFilter;
import android.content.IntentFilter.MalformedMimeTypeException;
import android.nfc.NfcAdapter;
import android.nfc.Tag;
import android.nfc.tech.MifareUltralight;
import android.os.Bundle;
import android.util.Log;
import android.widget.TextView;
import android.widget.Toast;

public class MainActivity extends Activity {

	private NfcAdapter mAdapter;
	private PendingIntent mPendingIntent;
	private IntentFilter[] mFilters;
	private String[][] mTechLists;
	private TextView mText;
	private int mCount = 0;

	private static final String TAG = MainActivity.class.getSimpleName();

	@Override
	public void onCreate(Bundle savedState) {
		super.onCreate(savedState);

		setContentView(R.layout.activity_main);
		mText = (TextView) findViewById(R.id.text);
		mText.setText("Scan a tag");

		onNewIntent(getIntent());
		mAdapter = NfcAdapter.getDefaultAdapter(this);

		if (mAdapter == null) {
			Toast.makeText(this, "Sorry, NFC is not available on this device",
					Toast.LENGTH_SHORT).show();
		} else {
			Toast.makeText(this, mAdapter.toString(), Toast.LENGTH_SHORT)
					.show();
		}

		// Create a generic PendingIntent that will be deliver to this activity.
		// The NFC stack
		// will fill in the intent with the details of the discovered tag before
		// delivering to
		// this activity.
		mPendingIntent = PendingIntent.getActivity(this, 0, new Intent(this,
				getClass()).addFlags(Intent.FLAG_ACTIVITY_SINGLE_TOP), 0);

		// Setup an intent filter for all MIME based dispatches
		IntentFilter filter = new IntentFilter(NfcAdapter.ACTION_TECH_DISCOVERED);
		try {
			filter.addDataType("*/*");
		} catch (MalformedMimeTypeException e) {
			throw new RuntimeException("fail", e);
		}
		mFilters = new IntentFilter[] { filter };

		// Setup a tech list for all NfcF tags
		mTechLists = new String[][] { new String[] { MifareUltralight.class
				.getName() } };
	}

	@Override
	public void onResume() {
		super.onResume();
		if (mAdapter != null)
			mAdapter.enableForegroundDispatch(this, mPendingIntent, mFilters,
					mTechLists);
	}

	@Override
	public void onNewIntent(Intent intent) {
        if (intent != null && intent.getAction().equals(NfcAdapter.ACTION_TECH_DISCOVERED)) {
 
                Bundle extras = intent.getExtras();
                Tag tag = (Tag) extras.get(NfcAdapter.EXTRA_TAG);
                final MifareUltralight ultralight = MifareUltralight.get(tag);
                
                mText.setText("Discovered tag " + ++mCount + " with intent: " + intent);
                String reader = readTag(tag);
        		Toast.makeText(this, reader, Toast.LENGTH_SHORT)
        		.show();
        } 
	}

	@Override
	public void onPause() {
		super.onPause();
		if (mAdapter != null)
			mAdapter.disableForegroundDispatch(this);
	}

	public String readTag(Tag tag) {
		MifareUltralight mifare = MifareUltralight.get(tag);
		try {
			mifare.connect();
			byte[] payload = mifare.readPages(4);
			return new String(payload, Charset.forName("ISO-8859-1"));
		} catch (IOException e) {
			Log.e(TAG,
					"IOException while writing MifareUltralight  message...", e);
		} finally {
			if (mifare != null) {
				try {
					mifare.close();
				} catch (IOException e) {
					Log.e(TAG, "Error closing tag...", e);
				}
			}
		}
		return null;
	}
}

Код я частично взял из документации и решил посмотреть, что получится. Запустил приложение, поднёс карточку метрок к задней крышке планшета Nexus 7 и увидел надпись: Discovered tag 1 with intent: Intent{act=android.nfc.action.TECH_DISCOVERED flg=0x20000000 cmp=ru.alexanderklimovnfctag./MainActivity(has extras)}, полученную через метод mText.setText().

Через всплывающее сообщение Toast я прочитал метку, которое отобразилось в виде кракозябр.

В принципе, я получил некоторое представление о технологии NFC. Практического применения для себя пока не вижу.

Кстати, если интересует более грамотный разбор информация с карточки метро, то можете на ГитХабе можете найти проект с открытыми исходниками https://github.com/ValleZ/Ticket-Info. Данный проект позволяет считывать информацию с карточки метро и определять число поездок, через какой турникет был проход, дату и многое другое. Еще следует обратить внимание на один момент - программа автоматически запускается, когда билет подносится к устройству. Постарайтесь самостоятельно разобраться, почему так происходит. В готовом виде приложение можно скачать в Google Play (https://play.google.com/store/apps/details?id=ru.valle.tickets). Судя по последнему отзыву, приложение теперь некорректно работает с новыми карточками метро, которые недавно ввели в Москве. Я не проверял. Для справки - приложение Яндекс.Метро также позволяет считывать информацию с карточки, но количество информации намного меньше.
