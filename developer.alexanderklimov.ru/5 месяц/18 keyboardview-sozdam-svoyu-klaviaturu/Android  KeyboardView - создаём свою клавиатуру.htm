KeyboardView - создаём свою клавиатуру

Закрытая зона/5-й месяц

Когда мы хотим ввести какие-то данные в текстовые поля, то обычно используем клавиатуры, которые установлены в системе. По сути, клавиатура - это обычное приложение в формате APK. Как правило, в системе уже установлена системная клавиатура, также можете стоять клавиатура от производителя аппарата, и можно загрузить отдельную клавиатуру с Google Play.

В большинстве случаев установленных клавиатур достаточно для ввода текста и чисел. Они обладают определённой гибкостью и могут подстраиваться под конкретное текстовое поле, выводя клавиши с цифрами, если нужно набрать телефонный номер и т.д.

Тем не менее, может возникнуть такая ситуация, когда вам понадобится свой вид клавиатуры. Допустим, мы хотим иметь клавиатуру для ввода шестнадцатиричных чисел с клавишами от 0-9 и A, B, C, D, F. Значит нам придётся разработать собственную клавиатуру.

Стоит отметить, что стандартные клавиатуры работают как сервисы и доступны для всех приложений. Мы же напишем упрощённый вид клавиатуры, который будет работать только в пределах нашей активности. Не стоит думать, что это половинчатое решение. Для нашей шестнадцатиричной клавитуры данный подход является оптимальным. Во-первых, не всем нужна такая клавитура и устанавливать лишний APK-файл пользователь не захочет. Во-вторых, текстовые поля не имеют атрибута для распознавания шестнадцатиричных чисел и мы не сможем автоматически выводить клавитуру. Поэтому, вполне логично внедрить нужную клавиатуру там, где она нужна.

В составе Android, начиная с API 3, есть класс KeyboardView - который наследуется от обычного View и является компонентом, который можно разместить на экране активности. Также можете ознакомиться в документации с классами Keyboard, Keyboard.Row и Keyboard.Key.

Начнём с разметки. Добавим на экран несколько текстовых полей и наш KeyboardView:

К сожалению, графический редактор Eclipse не распознаёт элемент KeyboardView и будет выводить сообщение об ошибке. Поэтому посмотреть, как будет выглядеть экран, у нас не получится.

Обратите внимание на установленные атрибуты. Мы позиционируем клавиатуру внизу родительского элемента (android:layout_alignParentBottom="true"), расширяем на всю ширину (android:layout_width="fill_parent") и убираем с экрана, чтобы она не мешалась (android:visibility="gone").

Заодно создадим ещё один файл в той же папке для отображения выбранной клавише в всплывающем окне:

res/xml/popup.xml


<?xml version="1.0" encoding="utf-8"?>
<Keyboard xmlns:android="http://schemas.android.com/apk/res/android"
    android:keyHeight="10%p"
    android:keyWidth="10%p" >

</Keyboard>

В коде клавиши из ресурсов подключаются к KeyboardView приблизительно так:


// подключаем клавиши из ресурсов
Keyboard mKeyboard= new Keyboard(MainActivity, R.xml.hexkbd);

// ссылка на KeyboardView
KeyboardView mKeyboardView= (KeyboardView)findViewById(R.id.keyboardview);
// присоединяем клавиши к компоненту
mKeyboardView.setKeyboard( mKeyboard );
// не выводим всплывающее окно для выбранной клавиши
mKeyboardView.setPreviewEnabled(false);

Возвращаясь к файлу hexkbd.xml, посмотрите на используемые атрибуты, которые определяют размеры, текст и т.д. Надписи на клавишах можно задать в виде строки (android:keyLabel) или ресурса Drawable (android:keyIcon).

Когда пользователь нажимает клавишу, то KeyboardView вызывает метод onKey() слушателя OnKeyboardActionListener. Нам следует задать необходиые коды нажатий в xml-файле, чтобы система понимала, какой символ следует выводить на экран.

Для клавиш, которые не имеют кода, например, перемещение курсора, нужно создать свои собственные коды, начиная с числа 55000 и выше.

Если мы забыли установить код для определённого символа, то система попытается вызвать нужный символ, считывая атрибут android:keyLabel. Если в атрибуте находится слово, то используется его первый символ. В других ситуациях выбрасывается исключение.

Атрибут android:isRepeatable="true" позволяет печатать символ многократно при долгом удерживании в клавиши нажатом состоянии. Такой подход хорош для перемещения позиции курсора.

Кстати, одной клавише можно присвоить список кодов (не больше двенадцати), например, android:codes="65, 66, 67". Это сделано для ввода альтернативных символов.

Всплывающее окно (popup.xml) выводится при длительном нажатии на клавишу. Например, при долгом нажатии на символ 'e' обычно выводятя символы на выбор: 'é', 'è', 'ë'.

Есть два способа для вывода всплывающего окна.

Можно прописать всплывающее окно в файле ресурсов клавиатуры, добавив атрибут android:popupKeyboard="@xml/newkeyboard" нужным клавишам. В этом случае придётся создать отдельный файл newkeyboard.xml, в котором следует описать клавиатуру.

Второй способ - добавить атрибут android:popupCharacters="aA" для клавиши 'A' и т.д, а также создать файл popup.xml, который был описан выше.

Программная часть

Для удобства создадим отдельный класс, в котором будет сосредоточена логика работы клавиатуры.

Нам нужно создать клавиатуру из ресурсов (класс Keyboard), присоединить её к KeyboardView и установить слушатель нажатий клавиш OnKeyboardActionListener. У слушателя есть несколько методов. Мы переопределим только метод onKey().

Если фокус получить компонент EditText, то следует работать с ним, в других случаях нужно игнорировать компонент и не использовать клавиатуру.

Также следует задать константы для собственных нестандартных клавиш.

Также нужно заставить систему не выводить стандартную клавиатуру в тех случаях, когда мы хотим показать свою клавиатуру и наоборот.

Если EditText имеет фокус, а клавиатура скрыта (например, после нажатия на клавишу Спрятать клавиатуру), то повторное нажатие в текстовом поле снова покажет клавиатуру.


edittext.setOnClickListener(new OnClickListener() {
    @Override public void onClick(View v) {
        showCustomKeyboard(v);
    }
});

Позаботимся о скрытии стандартной клавиатуры в момент загрузки приложения.


// Hide the standard keyboard initially
getWindow().setSoftInputMode(WindowManager.LayoutParams.SOFT_INPUT_STATE_ALWAYS_HIDDEN);

Также создадим код для обработки нажатия кнопки Back, чтобы она закрывала нашу клавиатуру (если она открыта).


@Override public void onBackPressed() {
    if( isCustomKeyboardVisible() ) hideCustomKeyboard(); else this.finish();
}

Если мы хотим выводить свою клавиатуру только для определённых текстовых полей, а для остальных использовать стандартную клавиатуру, то следует сделать небольшую проверку по идентификатору.


public void registerEditText(int resid) {
    // Find the EditText 'resid'
    EditText edittext= (EditText)findViewById(resid);
    ...
}
