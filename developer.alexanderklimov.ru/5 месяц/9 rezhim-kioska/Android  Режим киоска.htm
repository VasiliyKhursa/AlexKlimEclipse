Режим киоска

Для начала поясню, что имется в виду под словами "Режим киоска". Предположим, вы предоставляеете телефон в руки пользователя, но не хотите, чтобы он мог лезть на другие экраны из вашего приложения. Это может быть презентация на выставке, в магазине и т.д. Пользователь может только просматривать с телефона ваш сайт и ничего больше.

Базовая настройка

Приложение будет реализовано на основе компонента WebView, поэтому сразу установим разрешение на подключение к интернету:


<uses-permission android:name="android.permission.INTERNET"/>

Разметка будет очень простой:


<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent" >

    <WebView
        android:id="@+id/webView"
        android:layout_width="fill_parent"
        android:layout_height="fill_parent" />

</RelativeLayout>

Нам нужен браузер с ограниченными возможностями. Для этого нужно реализовать WebView со своим клиентом WebViewClient. WebViewClient - это специальный объект, определяющий поведение в различных ситуациях, которые происходят в браузере WebView, например, открытие по ссылке. Нам нужно использовать метод обратного вызова shouldOverrideUrlLoading() для нашей задачи. Также нам нужно запретить открытие ссылок из непредусмотренных мест. Об этом подробнее будет рассказано ниже. А пока создадим новый класс KioskWebViewClient.


package ru.alexanderklimov.kioskmode;

import android.webkit.WebView;
import android.webkit.WebViewClient;

public class KioskWebViewClient extends WebViewClient {
	final String mDomain = "alexanderklimov.ru";

	@Override
	public boolean shouldOverrideUrlLoading(WebView view, String url) {
		if (url.contains(mDomain)) {
			view.loadUrl(url);
		}
		return true;
	}
}

В классе мы задали строку для домена, внутри которого возможна загрузка страниц. Все внешние ссылки будут отсекаться.

Переходим к классу основной активности:


package ru.alexanderklimov.kioskmode;

import android.os.Bundle;
import android.app.Activity;
import android.view.Menu;
import android.webkit.WebView;

public class MainActivity extends Activity {

	private WebView mWebView;
	private final String mHomepageUrl = "http://developer.alexanderklimov.ru";

	@Override
	protected void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);
		setContentView(R.layout.activity_main);

		mWebView = (WebView) findViewById(R.id.webView);
		mWebView.setWebViewClient(new KioskWebViewClient());
		mWebView.loadUrl(mHomepageUrl);
	}
}

Вы можете настроить множество параметров у WebView через WebSettings:


WebSettings ws = mWebView.getSettings();

На этом базовая настройка приложения закончена. Теперь доведём его до ума.

Полноэкранный режим

Первое, что приходит в голову - сделать полноэкранный режим. Для этого нужно удалить заголовок и строку состояния. Перед вызовом метода setContentView() поместим строку кода, которая решает данную задачу:


// полноэкранный режим
requestWindowFeature(Window.FEATURE_NO_TITLE);
getWindow().addFlags(WindowManager.LayoutParams.FLAG_FULLSCREEN);

Кнопка Back

Следующий шаг - не дать пользователю возможность закрыть приложение через кнопку Back (Назад). Для этого нужно переопределить поведение метода активности onBackPressed(). Любая попытка нажать на эту кнопку приведёт к загрузке стартовой страницы.


@Override
public void onBackPressed() {
	mWebView.loadUrl(mHomepageUrl);
}

Кнопки громкости

Также можно заблокировать кнопки громкости. Для этого проверяются нажатия определённых клавиш:


private final List<Integer> mBlockedKeys = new ArrayList<Integer>(Arrays.asList(
		KeyEvent.KEYCODE_VOLUME_DOWN, KeyEvent.KEYCODE_VOLUME_UP));

@Override
public boolean dispatchKeyEvent(KeyEvent event) {
	if (mBlockedKeys.contains(event.getKeyCode())) {
		return true;
	} else {
		return super.dispatchKeyEvent(event);
	}
}

Кнопка Home

В обычных приложениях не рекомендуется блокировать кнопку Home (Домой), но для нашего приложения это необходимо. Следует отметит, что данный способ годится только для старых устройств до Android 4.0. У меня нет старых телефонов, поэтому данный способ на устройствах не проверял. А на эмуляторе данный код вызывает ошибку. Оставлю для общей картины.


// Для старых версий до Android 4.0
@Override
public void onAttachedToWindow() {
	getWindow().setType(WindowManager.LayoutParams.TYPE_KEYGUARD);
	super.onAttachedToWindow();
}

К счастью, есть другой способ. Мы можем назначить наше приложение как домашний экран устройства по умолчанию. В этом случае, когда пользователь будет нажимать на кнопку Home, он будет всё-равно попадать на наш экран.

Для реализации данной задачи нужно зарегистрирвать наше приложение через манифест, указав соответствующие настройки. Добавьте новый фильтр для своей активности:


<intent-filter>
    <action android:name="android.intent.action.MAIN" />

    <category android:name="android.intent.category.HOME" />
    <category android:name="android.intent.category.DEFAULT" />
</intent-filter>

Наше приложение теперь будет готово работать в качестве стартового экрана. При первой попытке нажатия на кнопку Home система попросит вас выбрать нужное приложение для этой цели. Если вы выберите вариант Всегда, чтобы выбор оставался по умолчанию, то диалоговое окно появляться не будет.

При каждом нажатии на кнопку Home создаётся новый экземпляр активности. Чтобы избежать этой проблемы, нужно указать, что активность должна запускаться в режиме singleTask. Режим также устанавливается через манифест:


<activity
    android:name="ru.alexanderklimov.kioskmode.MainActivity"
    android:label="@string/app_name"
    android:launchMode="singleTask" >

Включённый экран

Во время презентации выключение подсветки экрана не требуется. Этого эффекта также можно достичь программно. Добавим новый флаг в код, где мы устанавливали полноэкранный режим:


getWindow().addFlags(WindowManager.LayoutParams.FLAG_KEEP_SCREEN_ON);

Теперь выключить экран можно через кнопку питания телефона.Экран блокировки

Последний шаг - настроить так, чтобы ваше приложение было видимым всё время. Если телефон выключить и включить снова, то сразу появится ваше приложение, минуя экран блокировки, что достаточно удобно. Для этого добавляем новые флаги для окна активности:


getWindow().addFlags(WindowManager.LayoutParams.FLAG_DISMISS_KEYGUARD);
getWindow().addFlags(WindowManager.LayoutParams.FLAG_SHOW_WHEN_LOCKED);

Заключение

Итак, мы получили приложение, работающее в режиме Киоск. Передвижение по ссылкам достуно только в пределах заданого домена, все внешние ссылки блокируются. Также блокируются все кнопки и экран. Что ещё нужно для полного счастья?
