Третий месяц обучения (закрытая зона)

Если вам небходимо добавить в приложение поисковый механизм, то Android готова помочь вам, предлагая готовые решения в виде специальных диалоговых окон поиска или виджетов. Пользователь можете ввести нужный запрос и вы можете обработать данный запрос и вывести нужный результат. Причем вам также доступен голосовой поиск, поисковые подсказки на основе ранних запросов, поиск по данным вашего приложения.

Есть несколько вариантов для реализации поискового интерфейса. У них много общего, различается только реализация интерфейса.

Первый способ - использование системного поискового диалогового окна, который выводится в верхней части приложения. Система контролирует все события в поисковом диалоге. Когда пользователь отправляет запрос, система обрабатывает его и отдает результат указанной активности. Также система позволяет обрабатывать запрос по мере ввода текста.

Второй способ - использование виджета SearchView, который можно расположить в нужном месте вашего приложения. Напомню, что SearchView стал доступен, начиная с версии Android 3.0 (API Level 11).

Общий принцип работы поиска прост - когда пользователь делает запрос, то система создаёт намерение и сохраняет запрос в нём. Затем запускается указанная активность для обработки запроса, которому и передаётся намерение.

Вам необходимо настроить поисковый механизм. Он состоит из нескольких шагов:

  • Создание XML-файла, в котором содержатся различные параметры для поиска, включая настройки голосового поиска, подсказок и дополнений
  • Поисковая активность - форма, которая получает поисковый запрос, ищет данные и выводит результат
  • Поисковый интерфейс - обычно поисковая строка скрыта и выводится при нажатии на кнопку Поиска (если она есть на устройстве) или обычную кнопку, которая выполняет её функцию. Также используется виджет SearchView, который можно использовать в любом месте приложения, в том числе и в ActionBar

Создание простого приложения с поиском

Первый шаг - создаётся XML-файл для настройки обработки поисковых запросов (searchable configuration). В нём указываются различные пользовательские элементы. Обычно файлу дают название searchable.xml. Созданный файл помещают в папку res/xml/.

Файл должен содержать корневой элемент <searchable> с несколькими атрибутами:


<?xml version="1.0" encoding="utf-8"?>
<searchable xmlns:android="http://schemas.android.com/apk/res/android"
    android:hint="@string/search_hint"
    android:label="@string/app_name" >
</searchable>

Атрибут android:label является обязательным и указывает на имя вашего приложения. Второй атрибут android:hint не являетя обязательным, но тем не менее настоятельно рекомендуется к использованию, так как даёт подсказку пользователю. Согласитесь, если пользователь увидит подсказку Введите имя пропавшего котёнка, то он не станет вводить в поле запрос типа Почему Путин краб или что будет если бросить лом в унитаз поезда.

Это два основных атрибута, которые мы будем использовать. Про остальные читайте в документации.

Второй шаг - создание поисковой активности, которая принимает запросы и возвращает результат поиска. Как уже было сказано выше, при выполнении запроса создаётся намерение с действием ACTION_SEARCH. Активность получает запрос из области EXTRA намерения, ищет среди ваших данных и возвращает найденный результат. Не забываем прописать созданную активность в манифесте.

Начнём с манифеста. Как обычно в манифесте создаётся элемент <activity>. В нём указываются параметры, связанные с намерением ACTION_SEARCH в элементе <intent-filter>, а также добавляется элемент <meta-data>:


<application ... >
    <activity android:name=".MySearch" >
        <intent-filter>
            <action android:name="android.intent.action.SEARCH" />
        </intent-filter>
        <meta-data android:name="android.app.searchable"
                   android:resource="@xml/searchable"/>
    </activity>
    ...
</application>

Элемент <meta-data> должен содержать атрибуты android:name с значением android.app.searchable и android:resource, который должен ссылаться на созданный нами XML-файл.

Обратите внимание, что в <intent-filter> не используется элемент <category>, как это мы видим в основной активности приложения. Система сама понимает, что речь идёт о поиске и не нуждается в подсказке.

Закончив с настройкой активности в манифесте, можно приступать к созданию самой активности, которая будет принимать запрос, искать ответ среди предоставленных источников данных и предоставлять найдённые данные обратно. Обычно поисковые ответы выводятся в виде списка, поэтому часто для активности используются возможности ListActivity.

Итак, создаем новый файл для активности, например, MySearch.java. Первая задача активности - получить запрос от поискового диалога или виджета, который поступает через намерение. Сам запрос содержится в строке, из которого мы должны извлечь этот самый запрос:


public class MySearch extends Activity {
	@Override
	protected void onCreate(Bundle savedInstanceState) {
		// TODO Auto-generated method stub
		super.onCreate(savedInstanceState);
		// если используется какая-то разметка для активности
		//setContentView(R.layout.search); 

		// Получаем намерение, убеждаемся, что имеет нужное действие и получаем запрос
		Intent intent = getIntent();
		if (Intent.ACTION_SEARCH.equals(intent.getAction())) {
			String query = intent.getStringExtra(SearchManager.QUERY); // содержит запрос
			Toast.makeText(MySearch.this, query, Toast.LENGTH_LONG).show();
		}
	}
}

Переменная query содержит строку запроса SearchManager.QUERY, которая в нашем упрощённом примере выводится в всплывающем сообщении.

Сам поиск данных зависит от характера вашего приложения. Например, искать можно в базе данных SQLite (рекомендуется FTS3). У Гугла есть пример Searchable Dictionary с использованием этой техники.

Также данные могут содержаться на внешнем сервере, в файле и другие варианты. Всегда рекомендуется возвращать результаты с использованием адаптеров, например, CursorAdapter.

Несколько слов о поисковой строке. По умолчанию она всегда скрыта и пользователь должен явно активировать её. На старых моделях телефонов есть отдельная кнопка поиска для активации поисковой строки. Но сейчас наметилась тенденция отказа от этой кнопки, поэтому вам необходимо в своём приложении добавить свою кнопку и вызвать метод onSearchRequested(). Но этого недостаточно для показа поисковой строки, сначала необходимо активности разрешить использовать поисковый механизм. У нас уже есть созданная активность MySearch.

Снова возвращаемся в файл-манифест и добавляем элемент <meta-data> внутри элемента <activity> основной активности. В <meta-data> используется атрибут android:value, в котором указывается имя класса поисковой активности, а в атрибуте android:name указывается значение android.app.default_searchable. Вот как выглядит манифест в полном виде для приложения с основной активностью SearchInterfaceActivity и поисковой активностью MySearch:

. 

<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="ru.alexanderklimov.search"
    android:versionCode="1"
    android:versionName="1.0" >

    <uses-sdk android:minSdkVersion="15" />
    

    <application
        android:icon="@drawable/ic_launcher"
        android:label="@string/app_name" >
        <activity
            android:name=".SearchInterfaceActivity"
            android:label="@string/app_name" >
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />

                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>

            <meta-data
                android:name="android.app.default_searchable"
                android:value=".MySearch" />
        </activity>
        <activity android:name=".MySearch" >
            <intent-filter>
                <action android:name="android.intent.action.SEARCH" />
            </intent-filter>

            <meta-data
                android:name="android.app.searchable"
                android:resource="@xml/searchable" />
        </activity>
    </application>

</manifest>

Вот теперь поисковую строку можно вызывать из основной активности. Далее после создания поискового запроса запускается поисковая активность с обработкой намерения.

Если приложение состоит из нескольких экранов, то элемент <meta-data> можно поместить в элемент <application>, чтобы не дублировать элемент в каждой активности.

Мы проделали долгий путь и теперь готовы запускать приложение с поиском. Коротко повторим предыдущие шаги: создаем поисковую активность, вносим настройки в манифест и при необходимости добавляем кнопку (или пункт меню) на главной активности.

Стоит отметить, что при вызове метода onSearchRequested() ваше приложение теряет фокус. Это может пригодится, например, чтобы поставить игру на паузу (переопределяем метод):


@Override
public boolean onSearchRequested() {
    pauseSomeStuff();
    return super.onSearchRequested();
}

Если пользователь отменяет запрос, то поисковая строка закрывается и активность вновь получает фокус. Момент закрытия поисковой строки можно отследить через setOnDismissListener() и/или setOnCancelListener().
