Несколько лет назад в одном издательстве вышла моя первая книга, посвящённая MS Agent, которая использовала технологии распознавания речи, преобразования текста в речь и многие другие продвинутые вещи. К сожалению, наработки Microsoft не получили распространения и были отправлены на свалку истории. Но, по отдельности данные технологии по-прежнему развиваются. Поэтому, узнав, что в устройствах Android также используется технология Text To Speech (TTS), я очень обрадовался, как встрече с старым другом. Начиная с версии 1.6, в Android SDK появился класс android.speech.tts.TextToSpeech.

Сразу предупрежу, что на Android технология преобразования текста в речь выглядит сырой. Выбор языков ограничен (английский, испанский, французский, немецкий, итальянский), русский язык в пакет не включен, хотя существуют отдельные движки от независимых производителей с русским языком.

Поскольку место для хранения данных на некоторых устройствах под управлением Android ограничено, языковые пакеты не всегда изначально присутствуют в системе. Прежде чем запускать движок TTS, рекомендуется проверить наличие этих языковых библиотек.

Базовый пример

Для базового примера достаточно текстового поля для ввода текста и кнопки для воспроизведения этого текста вслух:


package ru.alexanderklimov.tts;

import java.util.Locale;

import android.app.Activity;
import android.os.Bundle;
import android.speech.tts.TextToSpeech;
import android.util.Log;
import android.view.View;
import android.widget.Button;
import android.widget.EditText;

public class TTSActivity extends Activity implements
		TextToSpeech.OnInitListener {

	private TextToSpeech tts;
	private EditText edit;
	private Button button;

	/** Called when the activity is first created. */
	@Override
	public void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);
		setContentView(R.layout.main);

		tts = new TextToSpeech(this, this);

		edit = (EditText) findViewById(R.id.editText1);
		button = (Button) findViewById(R.id.button1);
	}

	@Override
	public void onInit(int status) {
		// TODO Auto-generated method stub
		if (status == TextToSpeech.SUCCESS) {

			int result = tts.setLanguage(Locale.US);

			if (result == TextToSpeech.LANG_MISSING_DATA
					|| result == TextToSpeech.LANG_NOT_SUPPORTED) {
				Log.e("TTS", "This Language is not supported");
			} else {
				button.setEnabled(true);
				sayWords();
			}

		} else {
			Log.e("TTS", "Initilization Failed!");
		}
	}

	@Override
	public void onDestroy() {
		// Don't forget to shutdown tts!
		if (tts != null) {
			tts.stop();
			tts.shutdown();
		}
		super.onDestroy();
	}

	public void onButtonClick(View v) {
		sayWords();
	}

	private void sayWords() {
		String text = edit.getText().toString();
		tts.speak(text, TextToSpeech.QUEUE_FLUSH, null);
	}
}

Методы

Рассмотрим методы TextToSpeech

Изменение языка: метод setLanguage()

TTS поддерживает несколько распространённых языков. Вы можете изменить язык при помощи метода setLanguage():

int result = tts.setLanguage(Locale.JAPANESE); // японский язык

Изменение тембра голоса: метод setPitch()

Вы можете изменить тембр голоса, используя метод setPitch(float pitch). По умолчанию значение равно 1.0, чем меньше значение, тем ниже голос.

tts.setPitch(0.6);

Изменение скорости: метод setSpeechRate()

Скорость воспроизведения речи можно установить с помощью метода setSpeechRate(float speechRate). По умолчанию значение равно 1.0, чем меньше значение, тем медленнее говорит устройство.

tts.setSpeechRate(0.6);

Произношение: метод speak()

У метода speak(String text, int queueMode, HashMap params) несколько параметров:

  • text - текст для воспроизведения
  • queueMode - одна из констант (TextToSpeech.QUEUE_FLUSH, если хочется, чтобы предыдущая фраза прерывалась и сразу начиналась следующая и TextToSpeech.QUEUE_ADD, если хочется, чтобы предыдущая фраза договорилась до конца только после этого началась следующая)
  • params - массив дополнительных параметров. Возможные параметры:
    • TextToSpeech.Engine.KEY_PARAM_STREAM — поток, в котором будет воспроизводиться звук
    • TextToSpeech.Engine.KEY_PARAM_UTTERANCE_ID — идентификатор фразы. Пригодится, если хочется обрабатывать событие окончания фразы, и при этом не запутаться в произносимых фразах.

Тишина: метод playSilence()

Метод playSilence(long durationInMs, int queueMode, HashMap params) проигрывает тишину в течение заданного времени. Параметры те же, что у метода speak().

Конец разговора: метод stop()

Метод stop() останавливает воспроизведение.

Освобождаем ресурсы: shutdown()

Не забывайте вызвать метод shutdown(), чтобы освободить ресурсы движка TTS

Сохранение в файл: метод synthesizeToFile()

Метод synthesizeToFile(String text, HashMap params, String filename) записывает синтезированную речь в файл. Параметры те же, что у метода speak().

Вставка речи из файла: метод addSpeech()

Методы addSpeech(String text, String filename) и addSpeech(String text, String packagename, int resourceId) позволяют вместо синтезированной речи воспроизводить заранее подготовленный файл.

Слушатель для события окончания фразы: метод setOnUtteranceCompletedListener()

Метод setOnUtteranceCompletedListener(TextToSpeech.OnUtteranceCompletedListener listener) задаёт слушателя для события окончания фразы. Метод работает, если инициализировать из onInit и если вы в вызов метода speak() передаете параметры (HashMap) среди которых обязательно есть TextToSpeech.Engine.KEY_PARAM_UTTERANCE_ID.

Мои настройки - метод areDefaultsEnforced()

Метод areDefaultsEnforced() позволяет узнать, установлен ли в настройках TTS флажок «Мои настройки».

В следующем листинге показано, как определить наличие установленной библиотеки TTS, инициализировать новый объект TextToSpeech и использовать его для разговора на британском варианте английского языка.


private static int TTS_DATA_CHECK = 1;
private TextToSpeech tts = null;
private boolean ttsIsInit = false;

private void initTextToSpeech() {
    Intent intent = new Intent(Engine.ACTION_CHECK_TTS_DATA);
    startActivityForResult(intent, TTS_DATA_CHECK);
}

protected void onActivityResult(int requestCode,
        int resultCode, Intent data) {
    if (requestCode == TTS_DATA_CHECK) {
        if (resultCode == Engine.CHECK_VOICE_DATA_PASS) {
            tts = new TextToSpeech(this, new OnInitListener() {
                public void onInit(int status) {
                    if (status == TextToSpeech.SUCCESS) {
                        ttsIsInit = true;
                            if (tts.isLanguageAvailable(Locale.UK) >= 0)
                                tts.setLanguage(Locale.UK);
                                tts.setPitch(0.8f);
                                tts.setSpeechRate(1.1f);
                                speak();
                    }
                }
            });
        } else {
            Intent installVoice = new Intent(Engine.ACTION_INSTALL_TTS_DATA);
            startActivity(installIntent);
        }
    }
}
private void speak() {
  if (tts != null && ttsIsInit) {
    tts.speak("Hello, Kitty", TextToSpeech.QUEUE_ADD, null);
  }
}
@Override
public void onDestroy() {
  if (tts != null) {
    tts.stop();
    tts.shutdown();
  }
  super.onDestroy();
}

Голосовые движки

В современных телефонах уже есть один или два встроенных движка, например, на моём Galaxy SII есть TTS от Android и Samsung. По сути, голосовые движки являются отдельными программами, которые можно установить отдельно. Ниже дана короткая справочная информация об известных TTS-движках под Android.

Pico
Стандартный TTS-движок, знает 5 языков, поставляется бесплатно. Говорит неплохо, русского языка нет. Есть в эмуляторе Android 4
eSpeak
Свободный TTS-движок. Знает очень много языков. Русский язык есть, но очень плохой
SVOX
Популярный движок. Есть бесплатная программа-оболочка и платные голоса, которыми можно управлять из этой оболочки. Голосов очень много, включая русский язык. Рекомендуется для приложений с русской речью
Loquendo
Качественный движок. Для английского языка есть голос Susan, русского языка нет

Доработка приложения

В Android 4.0 и на моём телефоне уже установлены TTS-движки. В реальных примерах необходимо проверить наличие движков перед преобразованием текста в речь.

Добавим глобальную переменную перед методом onCreate():


private int MY_DATA_CHECK_CODE = 0;

В методе onCreate() добавим код:


Intent checkTTSIntent = new Intent();
checkTTSIntent.setAction(TextToSpeech.Engine.ACTION_CHECK_TTS_DATA);
startActivityForResult(checkTTSIntent, MY_DATA_CHECK_CODE);

Добавим в класс метод onActivityResult() (и удалим строчку tts = new TextToSpeech(this, this); из onCreate()):


protected void onActivityResult(int requestCode, int resultCode, Intent data) {
	if (requestCode == MY_DATA_CHECK_CODE) {
		if (resultCode == TextToSpeech.Engine.CHECK_VOICE_DATA_PASS) {
			tts = new TextToSpeech(this, this);
		} else {
			Intent installTTSIntent = new Intent();
			installTTSIntent
					.setAction(TextToSpeech.Engine.ACTION_INSTALL_TTS_DATA);
			startActivity(installTTSIntent);
		}
	}
}

Если соответствующие библиотеки успешно установлены, обработчик onActivityResult() получит действие CHECK_VOICE_DATA_PASS.

Если данные для воспроизведения речи недоступны, запустите новую активность, используя действие ACTION_INSTALL_TTS_DATA из класса TextToSpeech.Engine. Тем самым вы инициируете установку недостающих движков.

Метод getEngines() - получить список движков

В Android 4.0 (API 14) можно получить список установленных движков через метод getEngines(), который возвращает список List<TextToSpeech.EngineInfo>. С помощью метода getDefaultEngine() можно получить используемый движок по умолчанию. Добавим на экран текстовую метку для вывода движка, используемого по умолчанию, и Spinner для вывода списка всех движков:


// глобальные переменные
TextView tvDefaultTTS;
Spinner spInstalledEngines;
TextToSpeech tts;
List<TextToSpeech.EngineInfo> listInstalledEngines;
List<String> listInstalledEnginesName;
String defaultTTS;
	
// в методе onCreate()
tvDefaultTTS = (TextView) findViewById(R.id.defaulttts);
spInstalledEngines = (Spinner) findViewById(R.id.installedengines);

tts = new TextToSpeech(this, this);
listInstalledEngines = tts.getEngines();
listInstalledEnginesName = new ArrayList<String>();

for (int i = 0; i < listInstalledEngines.size(); i++) {
	listInstalledEnginesName.add(listInstalledEngines.get(i).label);
}

adapter = new ArrayAdapter<String>(this,
		android.R.layout.simple_spinner_item, listInstalledEnginesName);
adapter.setDropDownViewResource(android.R.layout.simple_spinner_dropdown_item);
spInstalledEngines.setAdapter(adapter);

defaultTTS = tts.getDefaultEngine();
tvDefaultTTS.setText("Default TTS Engine: " + defaultTTS);

Возможности движка

В API 15 появился метод getFeatures, позволяющий узнать возможности движка. На эмуляторе я ничего не увидел. На телефоне не проверял. За основу взят предыдущий пример.


package ru.alexanderklimov.tts;

import ...

public class TTSActivity extends Activity implements
		TextToSpeech.OnInitListener {

	TextView tvDefaultTTS, tvTTSInfo;;
	Spinner spInstalledEngines;
	TextToSpeech tts;
	List<TextToSpeech.EngineInfo> listInstalledEngines;
	List<String> listInstalledEnginesName;
	String defaultTTS;

	private ArrayAdapter adapter;

	/** Called when the activity is first created. */
	@Override
	public void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);
		setContentView(R.layout.main);

		tvDefaultTTS = (TextView) findViewById(R.id.defaulttts);
		tvTTSInfo = (TextView) findViewById(R.id.ttsinfo);
		spInstalledEngines = (Spinner) findViewById(R.id.installedengines);

		tts = new TextToSpeech(this, this);
		listInstalledEngines = tts.getEngines();
		listInstalledEnginesName = new ArrayList<String>();

		for (int i = 0; i < listInstalledEngines.size(); i++) {
			listInstalledEnginesName.add(listInstalledEngines.get(i).label);
		}

		adapter = new ArrayAdapter<String>(this,
				android.R.layout.simple_spinner_item, listInstalledEnginesName);
		adapter.setDropDownViewResource(android.R.layout.simple_spinner_dropdown_item);
		spInstalledEngines.setAdapter(adapter);

		defaultTTS = tts.getDefaultEngine();
		tvDefaultTTS.setText("Default TTS Engine: " + defaultTTS);

		spInstalledEngines.setOnItemSelectedListener(myOnItemSelectedListener);

	}

	OnItemSelectedListener myOnItemSelectedListener = new OnItemSelectedListener() {
		@Override
		public void onItemSelected(AdapterView<?> parent, View iew,
				int position, long id) {

			// shutdown old tts
			if (tts != null) {
				tts.shutdown();
			}

			// switch to new tts
			EngineInfo selectedEngineInfo = listInstalledEngines.get(position);
			String engine = selectedEngineInfo.name;
			tts = new TextToSpeech(TTSActivity.this, TTSActivity.this, engine);

		}

		@Override
		public void onNothingSelected(AdapterView<?> arg0) {
			// TODO Auto-generated method stub
		}
	};

	private void updateTTSInfo() {
		String info = "";
		Locale locale = tts.getLanguage();

		if (locale != null) {
			info += "Country: " + locale.getDisplayCountry() + "\n"
					+ "Language: " + locale.getDisplayLanguage() + "\n";

			Set feature = tts.getFeatures(locale);

			String[] featureArray = new String[feature.size()];
			feature.toArray(featureArray);
			for (int i = 0; i < featureArray.length; i++) {
				info += "feature : " + featureArray[i] + "\n";
			}
		}
		tvTTSInfo.setText(info);
	}

	@Override
	public void onInit(int status) {
		updateTTSInfo();
	}

	@Override
	public void onDestroy() {
		// Don't forget to shutdown tts!
		if (tts != null) {
			tts.stop();
			tts.shutdown();
		}
		super.onDestroy();
	}
}

Говорящие часы

Создадим говорящие часы. Поместим на форму элемент AnalogClock. Наша цель - при щелчке по часам телефон должен сказать нам время. Эта функциональность пригодится девушкам, у которых часто спрашивают: "Девушка, а не подскажете который час?". Представляете, она достаёт телефон и тыкает пальчиком в циферблат и суровый мужской голос отвечает на вопрос. Что-то мы отвлеклись от темы.

Добавляем код в предыдущий пример


// глобальные переменные
AnalogClock clock;
private int mHour, mMinute;

// в методе onCreate
clock = (AnalogClock) findViewById(R.id.analogClock1);
clock.setOnClickListener(ClockOnClickListener);

// слушатель для щелчка по часам
private AnalogClock.OnClickListener ClockOnClickListener = new AnalogClock.OnClickListener() {
	@Override
	public void onClick(View v) {
		// TODO Auto-generated method stub
		final Calendar c = Calendar.getInstance();

		mHour = c.get(Calendar.HOUR_OF_DAY);
		mMinute = c.get(Calendar.MINUTE);

		String myTime = "Now is " + String.valueOf(mHour) + " Hour "
				+ String.valueOf(mMinute) + " Minute";
		
		edit.setText(myTime);
		tts.speak(myTime, TextToSpeech.QUEUE_FLUSH, null);
	}
};
