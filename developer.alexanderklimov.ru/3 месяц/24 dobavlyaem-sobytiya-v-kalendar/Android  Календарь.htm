Разрешения

Начиная с Android 4.0, у разработчиков появился доступ к API системного приложения Календарь. До этого приходилось использовать недокументированные вызовы функций.

Кроме того, Календарь появился и в эмуляторе под Android 4.0 и выше. Прежде чем вы будете проверять работу примера, необходимо сначала настроить Календарь и завести в нём хотя бы один аккаунт. Задача оказалась непростой для меня, пришлось искать решение проблемы в интернете. Сложность возникает на следующем экране.

Необходимо отметить флажками оба пункта, указать сервер m.google.com, а потом прокрутить экран немного вверх, чтобы увидеть кнопку Далее.

Открыть Календарь - ACTION_VIEW

Запусить системное приложение Календарь с нужной датой можно следующим образом:


// текущая дата
long startMillis = System.currentTimeMillis();

Uri.Builder builder = CalendarContract.CONTENT_URI.buildUpon();
builder.appendPath("time");
ContentUris.appendId(builder, startMillis);
Intent intent = new Intent(Intent.ACTION_VIEW).setData(builder.build());
startActivity(intent);

Если нужно задать другую дату, то вот вам кусок кода (к текущей дате прибавляется два месяца вперед):


Calendar calendar = new GregorianCalendar(); 
calendar.setTime(new Date()); 
calendar.add(Calendar.MONTH, 2); 
long startMillis = calendar.getTime().getTime();

Добавляем события в Календарь - ACTION_INSERT

Создадим новый проект для Android 4.0 и выше (я использовал 4.1).

Для того, чтобы запустить Календарь из своего приложения и добавить в него новое событие, необходимо вызвать намерение с параметром Intent.ACTION_INSERT. Далее устанавливается необходимый тип, а также необходимые EXTRA-данные для предстоящего события. Такими данными могут выступать заголовок события, местоположение, описание самого события и т.д.

Минимальный код для добавления события:


Intent calIntent = new Intent(Intent.ACTION_INSERT);
calIntent.setData(CalendarContract.Events.CONTENT_URI);
startActivity(calIntent);

Толку от такого кода немного, откроется белый экран. Необходимо задать параметры, относящиеся к дате - когда произойдёт событие, когда напомнить, когда прекратить напоминание.

Ознакомьтесь в документации с другими данными, которые можно применить к Календарю, например, уровень приватности и другие возможности.

Теперь сам код. Основная логика находится в обработчике нажатия кнопки:


package ru.alexanderklimoff.calendarapp;

import java.util.GregorianCalendar;

import android.os.Bundle;
import android.provider.CalendarContract;
import android.provider.CalendarContract.Events;
import android.app.Activity;
import android.content.Intent;
import android.view.View;

public class MainActivity extends Activity {

	@Override
	public void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);
		setContentView(R.layout.activity_main);
	}

	public void onClick(View v) {
		Intent calIntent = new Intent(Intent.ACTION_INSERT);
		calIntent.setType("vnd.android.cursor.item/event");
		calIntent.putExtra(Events.TITLE, "Дела по дому");
		calIntent.putExtra(Events.EVENT_LOCATION, "Моя квартира");
		calIntent.putExtra(Events.DESCRIPTION,
				"Поздравить кота с Днём рождения!");

		GregorianCalendar calDate = new GregorianCalendar(2013, 6, 27);
		calIntent.putExtra(CalendarContract.EXTRA_EVENT_ALL_DAY, true);
		calIntent.putExtra(CalendarContract.EXTRA_EVENT_BEGIN_TIME,
				calDate.getTimeInMillis());
		calIntent.putExtra(CalendarContract.EXTRA_EVENT_END_TIME,
				calDate.getTimeInMillis());

		calIntent.putExtra(Events.RRULE,
				"FREQ=WEEKLY;COUNT=10;WKST=SU;BYDAY=TU,TH");

		startActivity(calIntent);
	}
}

Обратите внимание, что данные не записываются сразу в Календарь, пользователь должен явно нажать на кнопку Готово для подтверждения. Не забывайте, что в Android отсчет месяцев идет с 0, поэтому для июля нужно выбирать значение 6, а не седьмой месяц.

Разрешения

При работе с календарём внутри своего приложения требуются два разрешения для записи и чтения:


<uses-permission
    android:name="android.permission.READ_CALENDAR">
  </uses-permission>
<uses-permission
     android:name="android.permission.WRITE_CALENDAR">
   </uses-permission>

В предыдущих примерах разрешения не требовались, так как мы запускали через намерение внещнюю программу Календарь и работали уже через неё.
