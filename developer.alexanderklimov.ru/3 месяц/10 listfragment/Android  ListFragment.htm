ListFragment

Третий месяц обучения (закрытая зона)

Самая распространённое использование фрагментов - списки и связанное с ним содержание. При использовании списков на планшете в альбомной ориентации справа оставалось слишком много пустого пространства. Фрагменты позволяют использовать данное пространство с пользой.

Представим себе ситуацию - у нас есть список ListView, содержащий ссылки. При щелчке на одной из ссылок мы открываем вторую активность, состоящую из какого-нибудь компонента: TextView или WebView. По сути, один экран у нас сменяется другим. Можно реализовать эту задачу по другому. На планшетах много места. Почему бы не расположить ListView и TextView на одном экране рядышком. И когда пользователь будет щелкать слева на элементе списка, то в правой части будет обновляться содержимое TextView. Такой подход нам знаком, например, при чтении электронных писем - слева список писем, а справа - содержание выбранного письма.

Рассмотрим пример использования ListFragment, который приспособлен для подобных задач. Пример будет универсальным. На телефоне с портретной ориентацией будут использоваться две отдельные активности, а в альбомной ориентацией - одна активность с двумя фрагментами.

Так выглядит работа приложения в обычном телефоне, где первый экран представляет собой список, а второй - подробное описание выбранного пункта списка:

Начнём с первого класса. Нам понадобится массив для списка. Когда-то мы изучали урок с ListActivity. Возьмём данные оттуда:


private final String[] catnames = new String[] { "Рыжик", "Барсик",
		"Мурзик", "Мурка", "Васька", "Томасина", "Бобик", "Кристина",
		"Пушок", "Дымка", "Кузя", "Китти", "Барбос", "Масяня", "Симба" };

Добавим методы onCreate() и onListItemClick():


@Override
public void onCreate(Bundle savedInstanceState) {
	// TODO Auto-generated method stub
	super.onCreate(savedInstanceState);
	
	ListAdapter myListAdapter = new ArrayAdapter<String>(getActivity(),
			android.R.layout.simple_list_item_activated_1, catnames);
	setListAdapter(myListAdapter);
}

@Override
public void onListItemClick(ListView l, View v, int position, long id) {
	// TODO Auto-generated method stub
	super.onListItemClick(l, v, position, id);
	Intent launchingIntent = new Intent(getActivity(),
			DetailsActivity.class);
	launchingIntent.putExtra("catname", catnames[position]);
	startActivity(launchingIntent);
}

Обратите внимание, что через намерение мы передаём ключ catname, который будет содержать имя выбранного кота.

Теперь добавим код для класса DetailsFragment:


package ru.alexanderklimov.listfragmentdemo;

import android.app.Fragment;
import android.content.Intent;
import android.graphics.Color;
import android.os.Bundle;
import android.view.LayoutInflater;
import android.view.View;
import android.view.ViewGroup;
import android.widget.TextView;

public class DetailsFragment extends Fragment {
	TextView catdescription;
	
    @Override
    public View onCreateView(LayoutInflater inflater, ViewGroup container,
            Bundle savedInstanceState) {
        // TODO Auto-generated method stub
        Intent launchingIntent = getActivity().getIntent();
        String catname = launchingIntent.getStringExtra("catname");
        View viewer = (View) inflater.inflate(R.layout.viewer, container, false);
        catdescription  = (TextView) viewer.findViewById(R.id.tvDescription);
        updateCatDescription(catname);
        return viewer;
    }

	public void updateCatDescription(String name) {
        if (name != null)
            if (name.equals("Рыжик")) {
                catdescription.setBackgroundColor(Color.RED);
                catdescription
                        .setText("Рыжая наглая морда. Любит поспать и поесть. Мышей не ловит. Обожает радистку Кэт за её красивое имя.");
            } else if (name.equals("Барсик"))
                catdescription.setBackgroundColor(Color.BLUE);
            else if (name.equals("Мурзик"))
                catdescription.setBackgroundColor(Color.MAGENTA);
    }
}

Здесь мы объявили намерение, которое ожидает данные с ключом catname. Получив данные, вызываем метод updateCatDescription(). Для пущей наглядности, при совпадении имени меняем цвет фона у текстовой области. В своём приложении вы можете добавить какой-то дополнительный текст.

Кстати, разметка activity_main.xml, которая создавалась в проекте по умолчанию, нам не понадобится. Можете смело удалять её. Класс MainActivity нам тоже не нужен, так как мы создали собственные классы для активностей - удаляем и этот класс.

Не забываем про манифест. Добавляем в него две новые активности и удаляем запись, связанную с MainActivity.

Добавим новую разметку для альбомной ориентации в папку res/layout-land. Создадим нужную папку и скопируем туда файл fragment_a.xml. Внесём в файл небольшие изменения:

fragment_a.xml (res/layout-land)

<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="horizontal" >

Здесь нужно помнить важную вещь. Так как оба фрагмента сейчас находятся в одной активности, то старый код, который запускал новую активность, теперь не будеть работать для альбомной ориентации. Чтобы избежать ошибки, необходимо в коде проверить, где находится фрагмент и принимать дальнейшее решение.

Возвращаемся к классу TitlesFragment и вносим изменения в метод onListItemClick():


@Override
public void onListItemClick(ListView l, View v, int position, long id) {
    // TODO Auto-generated method stub
    super.onListItemClick(l, v, position, id);
    
    DetailsFragment viewer = (DetailsFragment) getFragmentManager()
            .findFragmentById(R.id.viewers);
    
    if (viewer == null || !viewer.isInLayout()) { // если два фрагмента в двух активностях
        Intent launchingIntent = new Intent(getActivity(),
                DetailsActivity.class);
        launchingIntent.putExtra("catname", catnames[position]);
        startActivity(launchingIntent);
    } else { // если в одной активности два фрагмента
        viewer.updateCatDescription(catnames[position]);
    }
}

Транзакции фрагментов

Одна из главных особенностей фрагментов - возможность транзакций. Используя транзакции, вы можете добавлять, удалять или заменять фрагменты в нужный момент. Транзакции можно сохранять и использовать как стек при использовании кнопки Назад. Давайте немного модифицируем фрагмент fragment_a.xml в папке layout-land.


<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/fragment_container"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:orientation="horizontal" >

    <fragment
        android:id="@+id/titles"
        android:name="ru.alexanderklimov.listfragmentdemo.TitlesFragment"
        android:layout_width="100dp"
        android:layout_height="match_parent" >
    </fragment>

    <fragment
        android:id="@+id/viewers"
        android:name="ru.alexanderklimov.listfragmentdemo.DetailsFragment"
        android:layout_width="220dp"
        android:layout_height="match_parent" >
    </fragment>

</LinearLayout>

Мы задали идентификатор для корневого элемента LinearLayout, а также размеры ширины у двух фрагментов и у нас осталось место для третьего фрагмента, который сейчас создадим.

ThirdFragment.java


package ru.alexanderklimov.listfragmentdemo;

import android.app.Fragment;
import android.graphics.Color;
import android.os.Bundle;
import android.view.LayoutInflater;
import android.view.View;
import android.view.ViewGroup;
import android.view.ViewGroup.LayoutParams;
import android.widget.TextView;

public class ThirdFragment extends Fragment {
	TextView mtext;

	@Override
	public View onCreateView(LayoutInflater inflater, ViewGroup container,
			Bundle savedInstanceState) {
		// TODO Auto-generated method stub
		TextView mtext = new TextView(getActivity());
		mtext.setText("Добавлен третий фрагмент. 
 Щелкните кнопку Назад для возврата");
		mtext.setBackgroundColor(Color.MAGENTA);
		mtext.setLayoutParams(new LayoutParams(LayoutParams.MATCH_PARENT,
				LayoutParams.MATCH_PARENT));
		return mtext;
	}
}

Изменим код в методе onListItemClick() у фрагмента TitlesFragment:


@Override
public void onListItemClick(ListView l, View v, int position, long id) {
	// TODO Auto-generated method stub
	super.onListItemClick(l, v, position, id);

	DetailsFragment viewer = (DetailsFragment) getFragmentManager()
			.findFragmentById(R.id.viewers);

	if (viewer == null || !viewer.isInLayout()) { // если два фрагмента в
													// двух активностях
		Intent launchingIntent = new Intent(getActivity(),
				DetailsActivity.class);
		launchingIntent.putExtra("catname", catnames[position]);
		startActivity(launchingIntent);

	} else { // если в одной активности два фрагмента
		viewer.updateCatDescription(catnames[position]);
		if (position == 0) {
			// ссылка на фрагмент
			Fragment newFrag = new ThirdFragment();
			// начинаем транзакцию
			FragmentTransaction trans = getFragmentManager()
					.beginTransaction();
			// добавляем ссылку на фрагмент к существующей разметке
			trans.add(R.id.fragment_container, newFrag);
			// добавляем транзакцию в историю стека
			trans.addToBackStack(null);
			// завершаем транзакцию
			trans.commit();
		} else {
			// другой код
		}
	}
}

Теперь, если запустить приложение и выбрать первый элемент списка, то на экране отобразится третий фрагмент (динамически). Если нажать кнопку Назад, то фрагмент исчезнет.
