В состав системы Android входит приложение Галерея, предназначенное для просмотра картинок и фотографий, а также видео. Вы можете взаимодействовать с Галерей из собственного приложения, используя возможности и функциональность системного приложения.

Запустить галерею

Запустить Галерею можно следующим образом:


Intent intent = new Intent(Intent.ACTION_PICK);
intent.setType("image/*");
startActivityForResult(intent, 1);

Это общий код. В следующих примерах мы рассмотрим более продвинутые примеры.

Выбираем изображение из Галереи на SD-карточке

Предположим, вы не хотите реализовывать в своём приложении файловый менеджер для выбора картинки, а хотите воспользоваться готовой программой. Нет проблем. Добавим на форму кнопку для вызова Галереи и метку для вывода пути выбранного изображения.


<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="fill_parent"
    android:layout_height="fill_parent"
    android:orientation="vertical" >

    <Button
        android:id="@+id/butSelectImage"
        android:layout_width="fill_parent"
        android:layout_height="wrap_content"
        android:onClick="OnButtonClick"
        android:text="Выбрать картинку" />

    <TextView
        android:id="@+id/tvTargetUri"
        android:layout_width="fill_parent"
        android:layout_height="wrap_content" />

</LinearLayout>

При щелчке кнопки мы вызовем приложение Галерея через намерение. Когда Галерея запустится и пользователь выберет нужную картинку, то результат вернётся в наше приложение:


package ru.alexanderklimov.galleryapp;

import android.app.Activity;
import android.content.Intent;
import android.net.Uri;
import android.os.Bundle;
import android.view.View;
import android.widget.TextView;

public class GalleryAppActivity extends Activity {

	TextView textTargetUri;

	/** Called when the activity is first created. */
	@Override
	public void onCreate(Bundle savedInstanceState) {
		super.onCreate(savedInstanceState);
		setContentView(R.layout.main);

		textTargetUri = (TextView) findViewById(R.id.tvTargetUri);
	}

	@Override
	protected void onActivityResult(int requestCode, int resultCode, Intent data) {
		// TODO Auto-generated method stub
		super.onActivityResult(requestCode, resultCode, data);

		if (resultCode == RESULT_OK) {
			Uri targetUri = data.getData();
			textTargetUri.setText(targetUri.toString());
		}
	}
	
    public void OnButtonClick(View v) {
		// запускаем Галерею
		Intent intent = new Intent(Intent.ACTION_PICK,
				android.provider.MediaStore.Images.Media.EXTERNAL_CONTENT_URI);
		startActivityForResult(intent, 0);
	}
}

Обратите внимание, что мы задаём просмотр изображений на внешней SD-карточке (EXTERNAL_CONTENT_URI).

Загрузка изображения из Галереи

Но получить путь к картинке, да еще в непонятном формате, не слишком интересно. Давайте попробуем выбранную картинку вставить в ImageView нашего приложения. Добавим на форму ImageView и переделаем код:


ImageView imageView;
static final int GALLERY_REQUEST = 1;

public void onCreate(Bundle savedInstanceState) {
    ...
	imageView = (ImageView) findViewById(R.id.imageView1);
    ....
}

@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
	// TODO Auto-generated method stub
	super.onActivityResult(requestCode, resultCode, data);

	Bitmap galleryPic = null;

	switch (requestCode) {
	case GALLERY_REQUEST:
		if (resultCode == RESULT_OK) {
			Uri selectedImage = data.getData();
			try {
				galleryPic = Media.getBitmap(getContentResolver(),
						selectedImage);
			} catch (FileNotFoundException e) {
				// TODO Auto-generated catch block
				e.printStackTrace();
			} catch (IOException e) {
				// TODO Auto-generated catch block
				e.printStackTrace();
			}
			imageView.setImageBitmap(galleryPic);
			textTargetUri.setText(selectedImage.toString());
		}
	}
}

public void OnButtonClick(View v) {
	// запускаем Галерею  с показом изображений на SD-карте
	Intent intent = new Intent(Intent.ACTION_PICK,
			android.provider.MediaStore.Images.Media.EXTERNAL_CONTENT_URI);
	startActivityForResult(intent, GALLERY_REQUEST);
}

Намерение ACTION_PICK вызывает Галерею. При этом возвращается адрес URI, определяющий местоположение выбранного изображения. Для его получения используется метод getData(). Далее для преобразования URI-адреса в сооответствующий экземпляр класса Bitmap используется специальный метод Media.getBitmap(). И у нас появляется возможность установить изображение в ImageView при помощи setImageBitmap().

Чтобы отобразить в Галерее все изображения, включая и внутреннюю память, попробуйте код:


Intent intent = new Intent(Intent.ACTION_PICK);
intent.setType("image/*");
startActivityForResult(intent, GALLERY_REQUEST);

Можно упростить код и вообще не вызывать объект Bitmap, используя метод setImageURI():


@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
	// TODO Auto-generated method stub
	super.onActivityResult(requestCode, resultCode, data);

	switch (requestCode) {
	case GALLERY_REQUEST:
		if (resultCode == RESULT_OK) {
			Uri selectedImage = data.getData();

			textTargetUri.setText(selectedImage.toString());
			imageView.setImageURI(selectedImage);
		}
	}
}

Сравните с предыдущим примером - чувствуете разницу? Тем не менее, приходится часто наблюдать подобный избыточный код во многих проектах. Это связано с тем, что метод порой кэширует адрес и не происходит изменений. Рекомендуется использовать инструкцию setImageURI(null) для сброса кэша и повторный вызов метода с нужным Uri.

Неплохо бы освобождать ресурсы в методах onStop()/onPause(), чтобы избежать проблем с памятью (не проверял):


public void onStop() {
    super.onStop();
    if (currentImage != null) {
        currentImage.recycle();
        currentImage = null;
        System.gc();
    }
}
